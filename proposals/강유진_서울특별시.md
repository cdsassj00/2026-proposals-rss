# 2. 실전 AX 과제 기획서

1. 과제 개요

○ 과제명 : 서울시 시정 핵심사업 'AI 분석기획 지능형 자동화' sLLM 서비스 구축 ○ 추진배경 - 행정 비효율 해소: 연간 300여 건에 달하는 핵심사업 전수 검토 및 컨설팅 시, 분석인력, 사업담당자의 직접 기획으로 인한 과다한 행정낭비, 절차·시간 등 소요 - 지식 자산화 필요: 그간 축적된 고품질 분석 계획서, 데이터 자산(공공·민간 데이터, 데이터정의서)를 AI학습 자산화 및 서비스 개발에 활용 - AX 기반 마련: 서울시의 'AI Readable' 공문서 생산 체계와 연계하여, 분석과제화 기획 단계부터 AI가 개입하는 지능형 행정 시스템으로의 전환 ○ 목표 - 정량목표: 분석 기획서 작성 시간 50% 이상 단축 - 정성목표: 핵심사업 담당자의 분석 역량 상향 평준화
2. 현안 문제 및 데이터 현황
○ 문제정의 - 시정 핵심사업 담당자가 사업계획 및 현안에 따른 분석 필요·요구사항은 있으나, 어떤 데이터, 분석 방법론을 활용하여 해결할지 과제 기획에 어려움을 겪음 - 기존 서울시 내부 '분석기획' 챗봇 서비스는 단순 질의응답 수준으로, 실제 서울시의 내부 데이터 정의서나 과거 분석사례를 맥락있게 반영하지 못함. ○ 보유 데이터 - 분석기획서: 최근 3년간 핵심사업 분석을 과제화한 전문 분석 계획서 - 저장데이터: 빅데이터 저장소 내 민간·공공데이터, 데이터 정의서(md,xlsx) - 사업계획서: AI Readable 형태로 생산된 서울시 실·국별 핵심사업 사업계획서
3. 구현 방안
○ 기술적 접근 (RAG 기반 sLLM 모델) - Vector DB 구축: 분석기획서와 데이터 정의서를 임베딩하여 벡터 DB화 - RAG 최적화: 사용자가 입력한 핵심사업 계획서의 맥락을 파악하여, 유사한 과거 분석사례와 활용 데이터 리스트를 실시간으로 검색 및 결합 - sLLM 서빙: 행정망 내부에 구축된 sLLM을 통해 기획서 초안 자동 생성지원 ○ 행정 절차 및 협업 - 데이터전략과: 가명처리/보안, AI 모델링, 행망 sLLM 서비스 연계 추진 - 사업부서: AI Readable 사업 계획서 업로드 및 생성된 분석 기획서 검토·활용
