# 지원동기  
국립의과학지식센터에서 다년간 학술논문 데이터 관리 및 검색 시스템 기획 업무를 수행해왔습니다. 2025년 11월부터 LLM 기반 **Reference-Aware RAG 프로토타입**을 자체 구축 중이며, 후속 고도화 사업을 직접 기획·추진하고 있습니다. 프로토타입 개발 환경은 다음과 같습니다:  
- **개발 도구**: 안티그래비티 IDE, Docker  
- **데이터베이스**: Oracle 11g(원본), PostgreSQL(벡터)  
- **LLM**: vLLM+Qwen2.5-7B, PubMedBERT  
- **하드웨어**: RTX-3060(12G vRAM)  

기존 키워드 검색 방식의 한계를 극복하기 위해 오픈소스 LLM과 벡터 데이터베이스를 결합한 **의미 기반 검색·해설형 응답 시스템**을 설계 중입니다. 그러나 에이전트 설계, 멀티모달 처리, RAG 고도화 등 핵심 실무 역량을 체계적으로 심화할 기회가 부족했습니다. 본 과정의 **Python 모델링, RAG, 에이전트, 웹앱 개발** 커리큘럼은 현재 진행 중인 과제와 직결되어 교육 기간 중 즉각적인 현장 적용이 가능합니다.  

교육 수료 후에는 AI 챔피언으로서 센터 내 **AI 전환 표준 모델**을 수립하고, 타 공공 의료·학술기관으로 확산하는 **거점 리더** 역할을 수행하겠습니다.  
- **관련 교육 경험**: 한국보건복지인재원 디지털역량강화과정(14H) 수료 ('25.8월)  
  - AI·디지털 플랫폼 이해, 생성형 AI 활용법, 데이터 리터러시 실습, 디지털 포용 전략 등

# 경력 및 실적  
## AI·데이터 분석 프로젝트 경험  
- **[2022~2024] 의과학정보시스템 고도화**  
  - 학술논문, 연구성과물(보고서 등) 메타데이터(제목·초록·키워드·저자·발행연도) 수십만 건 정제·표준화  
- **[2025~현재] LLM 기반 자연어 검색 고도화(RAG 프로토타입 구축)**  
  - 오픈소스 LLM(Llama, Gemma, OpenAI gpt-oss)과 벡터 DB를 활용한 **Reference-Aware RAG 아키텍처** 직접 설계·구현  
  - 연구자 정보와 학술논문을 연계한 **하이브리드 검색 기능** 프로토타입 구축  

## AI·데이터 분석 관련 업무 경험  
- 텍스트 클리닝, 메타데이터 정규화, 인명-논문 매핑 등 **데이터 전처리 파이프라인** 직접 구축  
- 학술논문 및 연구자 정보 임베딩 → 벡터 인덱스 생성 → 질의-문서 매칭 → 문맥 확장 → LLM 응답 생성의 **전체 RAG 파이프라인** 운영 경험 보유  
- 연구자·연구주제 기반 검색 및 질의응답 **프로토타입 UI/UX** 기획·테스트 수행

# 실전 AX 과제 기획서  
## 1. 과제 개요  
- **과제명**: Reference-Aware RAG 기반 의과학 학술·연구자 맞춤형 지식검색 플랫폼 고도화  
- **목표**:  
  - (정량 목표) 의미 기반 검색 도입으로 연구자 정보 탐색 정확도 **30% 이상 향상**, 해설형 응답 제공으로 자료 접근·활용 시간 **25% 단축**, 연간 신규 공동연구 연계 건수 **20건 이상 창출**  
  - (정성 목표) 공공 의과학 분야 **AI 전환 표준 모델** 수립 및 유사기관 확산 기반 마련  

## 2. 현안 문제 및 데이터 현황  
- **문제정의**:  
  - 키워드 검색 → 결과 리스트만 제공, 논문 간 맥락·연관성 파악 불가  
  - 연구자 프로필·협업 네트워크·주요 성과 비가시화 → 공동연구 기회 발굴 어려움  
  - 정책 입안 시 최신 학술 근거 신속 활용 한계  
- **보유 데이터**:  
  - 비전자(단행본, 정기간행물, 보고서): **156,506건**  
  - 전자(보고서, 전자자원): **33,723건**  
  - 구축자원(원문, 학술논문(서지), 데이터, 박물자료, 교육영상 등): **2,871,039건**  

## 3. 구현 방안  
| 단계 | 주요 내용 | 적용 기술 |  
|------|-----------|-----------|  
| ① 데이터 파이프라인 | 기 구축 DB → 텍스트 클리닝 → 메타데이터 정규화 → 키워드 매핑 | Python(pdfminer, Tesseract), FastAPI |  
| ② 벡터DB 구성 | 논문·연구자 정보 임베딩 → 멀티-모달 하이브리드 인덱스 생성 | Sentence-Transformers, Milvus (벡터 DB) |  
| ③ RAG 파이프라인 | 질의 → Top-k 문서 매칭·연관도 스코어링 → 문맥 확장(논문+연구자) → LLM 해설형 응답 생성 | Llama/Gemma/vLLM (오픈소스 LLM), LangChain, RAG |  
| ④ UI/UX 프로토타입 | 연구자·연구주제 기반 검색창 + 질의응답 영역 + 논문-연구자 연관 네트워크 시각화 | React, Chart.js, FastAPI 백엔드 |  
| ⑤ 제도·협업 | AI 활용 데이터 거버넌스 지침 마련, 부서 간 협업 체계 구축, 개인정보·보안 준수 절차 정립 | 내부 정보보안 정책 연계, 공공 데이터 API 활용 |  

## 4. 현장 이행계획 및 기대효과  
- **이행계획**:  
  - ① 파일럿 고도화 (2026.05 ~ 2026.08): 본 과정 교육 내용 즉시 적용 → RAG 프로토타입 정확도·응답 품질 개선  
  - ② 부서 내 적용 및 서비스 구축 (2026.09 ~ 2026.12): 사용자 피드백 수집 → 모델 개선 → 서비스 기능 구축  
  - ③ 이용자 서비스 오픈 (2026.12 ~): 타 기관과 데이터 공유·협력, RAG 모델 이전·확산 지원  
- **기대효과**:  
  - **업무 효율**: 연구자 정보 탐색 정확도 30% 향상, 학술자료 접근·활용 시간 25% 단축  
  - **연구 성과**: 해설형 응답 제공으로 학제 간 협력 촉진, 연간 신규 공동연구 연계 20건 이상 지원  
  - **행정 혁신**: 근거 기반 정책 수립 지원, 외부 검색컨설팅 비용 절감  
  - **확산 가능성**: 공공 의료·학술기관 AI 전환 표준 모델 제공, 범정부 AI 지식검색 인프라 기여

국립의과학지식센터 건립부터 서비스 구축·운영 과정에서 학술논문 데이터 관리와 검색 시스템 기획·관리 업무를 수행했습니다. 2025년 말부터 LLM 기반 **Reference-Aware RAG 프로토타입**을 직접 기획·설계·구축 중이며, 이 과제는 공공 의과학 분야의 **AI 전환 표준 모델** 구축을 목표로 합니다.  

그러나 에이전트 설계, 멀티모달 처리, RAG 고도화, 웹앱 개발 등 핵심 실무 기술을 체계적으로 심화할 기회가 부족했습니다. 본 과정은 이러한 실전 역량을 단계적으로 습득하고, 진행 중인 과제에 즉시 적용할 수 있는 최적의 기회입니다.  

교육 중 개인·팀 프로젝트를 통해 현재 RAG 프로토타입을 고도화하고, 전문 컨설턴트와 협업해 실제 서비스(PoC)로 구현하겠습니다. 수료 후에는 AI 챔피언으로서 기관 내 AI 전환을 주도하고, 공공 의료·학술 분야 전반에 성공 모델을 확산하는 **거점 리더**가 되겠습니다.
