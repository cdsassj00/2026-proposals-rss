# 1. 지원동기  
저는 제어계측공학 전공 및 GPS 알고리즘 설계 석사 학위 취득 후, 기업 연구소에서 모터제어 연구를 수행한 공학도 출신 공무원입니다. 기술이 현실 문제를 해결하는 과정을 연구자로서 오랫동안 관찰해 온 저에게 AI는 공공 행정의 효율성을 혁신할 수 있는 강력한 도구로 다가왔습니다. 이 확신은 실제 구현과 학습으로 이어져, **n8n 기반 체납차량 최적경로 자동화 프로그램** 개발, **웹 크롤링 기반 AI 지방세 Q&A 시스템** 구축, **MCP 서버 재구현을 통한 답변 품질 향상** 등의 성과로 나타났습니다. 또한 AI 스터디와 Databus.kr 강좌를 통해 폐쇄망 LLM 구축, AI 에이전트 개발, 데이터 분석 등 다방면의 실전 경험을 쌓았습니다.  

공학적 배경과 지방세·지방행정에 대한 이해를 바탕으로 기술과 행정을 연결하는 가교 역할을 수행하고자 합니다. 본 교육과정을 통해 AI·데이터 분석 역량을 강화하여, 고령군을 시작으로 소규모 지자체에 적용 가능한 AI 모델과 실행 방안을 마련하고 주민들에게 실질적인 변화를 만들어가고자 합니다.

# 2. 경력 및 실적  
### 가. 웹 크롤링 기반 AI 지방세 Q&A 시스템 개발  
- **OLTA 데이터 크롤링** → 벡터 DB 적재 → FastAPI 백엔드 + React+Vite 프론트엔드 파이프라인 구현  
- **MCP 서버 재설계**로 Claude Code CLI 환경에서 답변 품질 향상 (단순 RAG 대비 문맥 이해 및 법령 해석 정확도 개선)  
- 지방세 실무 전문성과 AI 기술 결합의 대표적 성과  

### 나. n8n 기반 체납차량 최적경로 자동화 프로그램 개발  
- **Kakao Map API** (Geocoding) + **OpenAI Agent** (최적 경로 산출) + **네이버 길찾기 앱 딥링크** 통합 워크플로우 구현  
- 기존 30분 소요 작업을 1분 이내로 단축, 유사 업무 확산 가능 모듈 설계  
- 생성형 AI 활용 공모전 출품  

### 다. AI 스터디 및 체계적 자기계발  
- AI 챔피언 블루 인증자들과의 스터디에서 **폐쇄망 LLM 구축**, **AI 에이전트 개발**, **데이터 분석**, **AI 영상 제작** 등 실전 프로젝트 수행  
- Databus.kr 강좌 이수 및 AI 관련 서적 심화 학습  

### 라. 이공계 연구 배경 및 데이터 분석 역량  
- 제어계측공학 석사 과정에서 **신호처리**, **좌표 변환**, **최적화 알고리즘** 연구  
- 기업 연구소에서 데이터 기반 의사결정 및 시스템 설계 역량 확보  

### 마. 지방세 행정 전문성 및 수상 이력  
- 취득세, 등록면허세, 자동차세 법령 해석 및 감면 요건 검토 실무 수행  
- 2025년 지방세 발전포럼 3위 (행정안전부장관상), 2024~2025년 OLTA 활용대회 입상  
- 「지방세특례제한법」 등 법령 이해를 바탕으로 AI 시스템 설계 시 차별화된 강점 보유

# 3. 실전 AX 과제 기획서  
## 1. 과제 개요  
### 가. 과제명  
**"AI로 더 먼저 연결하고, 더 안전하게 보호하다" - 전국 확산형 주민 소통 플랫폼 "AI 읍장님"**  

### 나. 추진배경  
- 읍·면 단위 주민 소통은 현장 방문, 이장 회의 등 수동적 방식에 의존하며, 고령화로 인한 위기 신호 미감지 문제 발생  
- 통화 기록 미비로 정책 근거 데이터 활용 불가, 담당자 교체 시 노하우 단절  
- 행정 안내·복지 연계 수작업으로 인한 업무 과부하  

### 다. 목표  
#### 1) 정량적 목표  
| 항목 | 현재 | 목표 |  
|------|------|------|  
| 정기 안부 확인 주기 (독거노인) | 월 1회 | 주 1회 이상 |  
| 위기 감지 → 공무원 알림 소요시간 | 수일 ~ 수주 | 5분 이내 |  
| 행정 안내 전달률 | 30% 미만 | 80% 이상 |  
| 통화 기록·분석 체계화율 | 0% | 100% |  
| 담당 공무원 반복업무 시간 | 주당 15시간 | 주당 5시간 (67% 절감) |  

#### 2) 정성적 목표  
- 주민 체감 신뢰도 향상, 선제적 복지 행정 실현, 숨은 민원 사전 포착, 전국 확산 가능 표준 모델 수립  

## 2. 문제정의 및 데이터 현황  
### 가. 문제정의  
1) **소통 사각지대**: 물리적 한계, 디지털 접근성 부족  
2) **비체계적 기록·분석**: 수기 메모 의존, 담당자 교체 시 노하우 단절  
3) **반복 업무 과부하**: 수작업 프로세스, 개인 역량에 의존  

### 나. 보유 데이터  
| 데이터명 | 보유주체 | 활용 방안 |  
|----------|----------|-----------|  
| 주민등록 데이터 | 새올행정시스템 | 통화 대상자 선정 |  
| 독거노인·취약계층 현황 | 행복e음 | 통화 우선순위 산정 |  
| 복지서비스 제공 현황 | 행복e음 | 복지 맞춤 안내 |  
| 민원 접수·처리 현황 | 새올행정시스템 | 기존 민원 매칭 |  
| 행정 전달사항·시책 | 시·군·구청 | 통화 시 안내 콘텐츠 |  

## 3. 구현 방안  
### 가. 시스템 아키텍처  
- **3단계 역할 분담 구조**: n8n (대상자 선정) → Python 기반 AI 엔진 (실시간 대화) → n8n (사후 처리)  
- **PoC → 내부 서버형 → SaaS 확산** 로드맵  

### 나. AI·데이터 기술 구성  
| 기술구성 | 활용 기술 | 활용 방안 |  
|----------|-----------|------------|  
| 음성 인식(STT) | Whisper | 주민 음성 → 텍스트 변환 |  
| 대화 생성(LLM) | 폐쇄망 상용/오픈소스 모델 | 과거 이력·주민 정보·행정 전달사항 종합 응답 |  
| 음성 합성(TTS) | 공식 안내용 합성 음성 모델 | 응답 텍스트 → 음성 전달 |  
| 자동화 워크플로 | n8n | 대상자 선정, 전달사항 연계, 사후 분류·알림 |  
| 데이터 분류 | LLM 기반 멀티라벨 분류 | 대화 내용 → 6대 항목 자동 분류 |  

### 다. 제도 개선 및 행정 절차  
- 개인정보 보호: 통화 녹음 데이터 암호화 저장·자동 파기  
- 윤리 가이드라인: 특정 공직자 음성 복제 금지, 긴급 상황 시 공무원 후속 판단 의무화  
- 부서 협업: 읍·면 행정복지센터, 정보화 담당, 복지 담당 협업 체계 구축  

## 4. 현장 이행계획 및 기대효과  
### 가. 현장 이행계획  
| 단계 | 주요활동 | 성과지표(KPI) |  
|------|-----------|----------------|  
| 1단계 (POC 검증) | 대가야읍 50가구 대상 AI 안부 전화, 기술 검증 | 통화 성공률 80% 이상 |

### 나. 기대 효과

- 대국민 서비스 품질 향상 (선제적 복지 행정, 위기 조기경보 체계)  
- 전국 확산 가능성 (SaaS형 공동 운영 모델)
