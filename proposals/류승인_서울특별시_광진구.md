□ 세부사항

| 1. 지원동기 |
| --- |
| - 2025년 12월 AI챔피언 블루 인증 취득 이후, AI 기술의 공공 업무 적용 가능성을 직접 체감하며 다수의 AI 활용 프로젝트를 자체 기획·개발·운영. - 그 결과 국가인공지능전략위원회로부터 공직사회 디지털·AI 전환 모범 사례로 소개되었고, 헤럴드경제, 서울신문, 전자신문 등 주요 매체에 보도되는 성과를 거둠. - 이러한 경험을 통해 '만드는 역량'은 갖추었으나, 축적된 데이터를 분석하여 정책에 활용하는 체계적 방법론이 부족함을 절감. 본 과정을 통해 도구 개발자에서 데이터 기반 의사결정 설계자로 역할을 확장하고자 지원, |
| 2. AI·데이터분석 활용 경력 및 실적 |
| - 코닥(Kordoc): HWP·HWPX·PDF 등 공공기관 문서를 AI가 활용 가능한 형태로 변환·분석. 5개 공공 프로젝트에서 수천 건 문서로 성능 검증. GitHub 오픈소스 공개(기술: Python, 문서 파싱, AI 연동) - Korean-law-mcp: 법제처 API 기반 17만 5천여 건 대한민국 법령 통합 검색. 헌법·법률·행정규칙·판례 등 64개 구조화 도구 제공. AI 어시스턴트 연동. GitHub 오픈소스 공개(기술: TypeScript, MCP, 법제처 Open API) - 교육시설 AI 자문 (edu-facility-ai): 교육시설안전원 사전기획팀 AI 전환 자문. OCR 기반 문서 분류·태깅, 2단계 교육과정 분석, 7개 공공데이터 크롤러 구축 (기술: Tauri, Gemini API, Playwright) 법령비교 플랫폼 (Lexdiff): 국가법령↔자치법규 실시간 비교, AI 자연어 검색, 3단계 위임 체계 시각화.(기술: Next.js, Gemini RAG) - Anything (Docufinder): 로컬 문서(HWP, DOCX, PDF 등) 하이브리드 검색 엔진. 키워드(FTS5) + 의미(벡터 임베딩) 검색을 오프라인에서 지원 (기술: Tauri, ONNX Runtime, SQLite FTS5) |
| 3. 데이터 분석·AI활용 계획 |
| ○ 현재 운영 중인 AI 도구들을 데이터 기반 의사결정 체계로 고도화 법령 데이터 활용 확대: Korean-law-mcp와 lexdiff를 연계하여 자치법규 제·개정 시 유사 조례 비교 분석, 법령 충돌 자동 탐지 시스템 구축 - 문서 분석 자동화: Anything의 벡터 검색 엔진을 기관 내부 문서(기안문, 보고서)에 적용하여 유사 업무 사례 자동 추천 시스템 개발 - 민원 처리 이력 데이터 분석: 민원 유형별 빈도·처리시간 패턴 분석을 통한 선제적 민원 대응 체계 마련 |

# 2. 실전 AX 과제 기획서

| 1. 과제 개요 |
| --- |
| ○ 과제명: 기관 내부 문서 AI 검색 및 유사 업무사례 자동 추천 시스템 "Anything" 구축 ○ 추진배경 - 연간 수천 건 내부문서(기안문 보고서·지침) 생산 - 전자문서시스템 검색: 제목·키워드 단순 매칭에 한정 - 유사 선례 탐색 시 구두 문의 또는 수십 페이지 수동 열람 필요 - 신규 전입·인수인계 시 업무 맥락 파악에 과도한 시간 소요 ○ 목표 - (정량) 업무 선례 검색 소요시간 70% 단축 (평균 30분 → 10분 이내) - (정량) 신규 전입자 업무 적응 기간 40% 단축 - (정성) 부서 간 업무 지식 단절 해소, 조직 학습 역량 강화 |
| 2. 현안 문제 및 데이터 현황 |
| 문제정의 검색의 한계: 현행 시스템은 정확한 키워드를 알아야만 검색 가능. "작년에 비슷한 민원 처리한 사례"처럼 맥락 기반 검색 불가 - 문서 형식의 다양성: HWP, HWPX, DOCX, PDF, 스캔 이미지 등 다양한 포맷이 혼재하여 통합 검색 불가 - 지식의 개인화: 업무 노하우가 담당자 개인에게 편중되어, 인사이동 시 업무 공백 발생 ○ 보유 데이터 - 기관 내부 전자문서 (기안문, 보고서, 지침류): 연간 약 5,000~10,000건 (HWP/HWPX 중심) - 코닥(Kordoc)으로 변환·파싱 가능한 문서 포맷 전체 지원 확인 완료 - 자치법규 및 조례 데이터: 법제처 Open API 통해 실시간 연동 가능 - 민원 처리 이력 데이터: 기관 민원시스템에서 추출 가능 (CSV/DB) 3. 구현 방안 |
| ○ 기술적 접근 |
| - 문서 수집·전처리: 코닥(Kordoc)을 활용하여 HWP/HWPX/DOCX/PDF 텍스트 추출, 스캔 문서는 OCR(PaddleOCR) 적용. 이미 5개 공공 프로젝트에서 수천 건 문서로 검증된 파이프라인 활용 - 하이브리드 검색 엔진 구축: · 키워드 검색: SQLite FTS5 기반 한국어 형태소 분석 |

| · 의미 검색: KoSimCSE 임베딩 + 벡터 유사도 검색 (usearch) · Cross-Encoder 리랭킹으로 검색 정확도 향상 - 유사 업무사례 추천: 문서 간 유사도 매트릭스를 사전 계산하여, 현재 작성 중인 기안문과 유사한 과거 사례를 자동 추천 - AI 질의응답: Gemini API 기반 RAG 구성으로 "이 업무 관련 지침이 뭐야?" 같은 자연어 질문에 근거 문서와 함께 답변 제공 ○ 제도적 접근 - 유관부서와 협업하여 문서 인덱싱 자동화 파이프라인 구축 (신규 결재 문서 자동 색인) - 개인정보·보안 등급별 접근 권한 설계 (기존 전자문서시스템 권한 체계 연동) - 부서별 파일럿 운영 → 피드백 수렴 → 전 부서 확산 단계적 추진 |
| --- |
| 4. 현장 이행계획 및 기대효과 |
| 현장 이행계획 - 1단계(PoC): 소속 부서 문서 대상 프로토타입 구축. 기존 Anything·코닥 코드베이스 활용하여 빠른 검증(1~2개월) - 2단계(파일럿): 2~3개 부서로 확대 적용. 사용자 피드백 기반 검색 정확도 튜닝, UI/UX 개선(3~4개월) - 3단계(확산): 전 부서 적용. 정보화담당관실 연계 자동 색인 파이프라인 안착, 사용자 교육 실시(5~6개월) - 4단계(고도화): 타 지자체 공유 모델 개발, 검색 패턴 분석 → 업무 프로세스 개선 인사이트 도출(7개월~) ○ 기대 효과 - 업무 시간 단축: 문서 검색·선례 조사 시간 70% 절감 (연간 약 1,500시간 절감 추정, 300명 기준 1인 5시간/월) - 업무 품질 향상: 과거 유사 사례 참조율 증가로 기안문·보고서 품질 표준화 - 조직 지식 보존: 인사이동에 따른 업무 공백 최소화, 신규 전입자 온보딩 기간 단축 - 확산 가능성: 오프라인 작동(인터넷 불요) 설계로 보안 환경의 타 공공기관 즉시 적용 가능. 코닥·Korean-law-mcp와 함께 오픈소스 공개 예정 |

# □ AI · 데이터 거점리더 활동계획

| 1. 데이터기반행정 거점 리더 활동 계획 |
| --- |
| ○ AI 인재양성 및 문화확산: - 비전공자 대상 AI 교육 및 동호회 활동 지속 (월 2회 이상) - 기관 내 "AI 점심 세미나" 운영: 실제 업무 적용 사례 중심 30분 실습형 교육 - AI챔피언 블루 이수 경험 기반 기관 내 AI 멘토링 체계 구축 ○ 데이터기반 정책 수립·제도 마련: - Korean-law-mcp를 활용하여 자치법규 제·개정 시 데이터 기반 비교 분석 프로세스 도입 - Anything 운영 데이터로 부서별 업무 패턴 분석 → 업무 프로세스 개선 제안 ○ 외부 데이터 연계 및 API 활용 계획: - 법제처 Open API, 공공데이터포털, Kakao Local API 등 공공 API 활용 가이드 제작·공유 - 자체 개발 AI 도구를 오픈소스로 공개하여 타 기관 활용 지원 (코닥, Korean-law-mcp 이미 공개 완료) - 타 지자체 AI 담당자와 데이터 공유· 협력 네트워크 구축 |
| 2. 과정 참여 필요에 대해 자유롭게 기술 |
| AI챔피언 블루 과정 이후 23개 이상의 개인 AI 프로젝트를 직접 개발하고, 그 중 코닥과 Korean-law-mcp는 국가인공지능전략위원회 혁신 사례로 소개되어 전자신문에 보도되었습니다. 경영학과 출신 비전공자로서 독학으로 이 성과를 만들었다는 점은, AI 기술이 전공 장벽 없이 공직 현장에서 실질적 가치를 창출할 수 있다는 증거라고 생각합니다. ○ 그러나 솔직히 한계도 명확합니다. 현재까지의 프로젝트는 "만들기"에 집중되어 있고, 축적된 데이터를 분석하여 정책 의사결정에 활용하는 역량은 부족합니다. 도구는 있는데 그 도구가 만들어낸 데이터를 제대로 읽어내지 못하는 셈입니다. ○ 본 과정을 통해 체계적인 데이터 분석 방법론을 습득하고, 도구 개발자에서 데이터 기반 의사결정 설계자로 역할을 확장하고 싶습니다. 이미 검증된 도구 인프라(코닥, Anything, Korean-law-mcp) 위에 분석 역량이 더해진다면, 기관 내 AI 전환을 넘어 타 공공기관으로의 확산까지 이끌 수 있는 거점 리더가 될 수 있다고 확신합니다. |
