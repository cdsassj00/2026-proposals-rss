# 지원서 정리 문서

## 1. 지원동기  
### 가. 참여동기 및 기대효과  
- 생성형 AI(ChatGPT) 활용 데이터 분석 자동화 도구 개발, 교육행정데이터통합관리시스템(교데통) 데이터시각화 사용자매뉴얼 제작, 통합 AI 매뉴얼 챗봇 구축, 부서별 데이터 목록화 및 공유 확대 업무를 직접 추진하며, AI와 데이터 분석이 실제 행정업무 개선에 활용될 수 있음을 확인  
- 현재 실무 경험과 자체 학습을 바탕으로 업무를 추진하고 있으나, 데이터 분석 방법론, AI 활용 설계, 과제 구체화 측면에서 체계적인 보완이 필요하다고 판단  
- 현장에서 반복적으로 발생하는 데이터 취합·정리·표준화 업무, 사용자 문의 대응, 부서 간 데이터 공유 문제를 실행 가능한 과제로 발전시키기 위해 본 과정 참여 신청  
- 교육 이수 후 현재 추진 중인 자동화 도구 개발, AI 기반 사용자 지원, 데이터 공유 확대 업무에 교육 내용을 적용하여 실무에 바로 활용 가능한 방식으로 발전시킬 계획  

### 나. 관련교육 경험  
- **(행정안전부 AI챔피언 블루 과정 수료 및 인증)**  
  - AI 전환사업 기획·설계, PoC부터 구축·준공까지 실무 전 과정을 이끄는 실행 중심 역량 인증  
  - 생성형 AI 활용, 데이터 분석, 서비스 구현 등 공공행정 문제해결형 AI 역량 보완

## 2. 경력 및 실적  
### 가. AI·데이터 분석 프로젝트 경험  
#### 1) 생성형 AI(ChatGPT) 활용 데이터 분석 자동화 도구 개발  
- **추진 배경**  
  - 엑셀 병합, 데이터 정리, 표준화, 통계 산출 등 반복적 데이터 처리 업무 비중 높음  
  - 수작업 중심 처리로 시간 소요 및 실수 가능성 존재  
  - 현장에서 바로 사용 가능한 도구 필요  

- **기획 및 설계**  
  - 2025.11.부터 생성형 AI(ChatGPT) 활용 협업형 내부 개발 방식 추진  
  - 초기 개발 범위: 엑셀 병합, 데이터 품질 점검, 학교명 표준화, 통계 요약 리포트  
  - 입력-처리-출력 구조로 기능 요구사항 정리 및 사용자 흐름·화면 구성 설계  

- **개발 및 보완**  
  - 생성형 AI로 기능별 코드 초안, 사용자 안내문, 오류 메시지 초안 작성  
  - 기능 확대: 형태 변환·언피벗 도구, 엑셀 분할, 파일명 정리, 이수증 PDF 자동 취합 도구 등  
  - 1차·2차 검토의견 반영하여 초기화, 상태 리셋, 다운로드 안정화, 날짜·유형 검증, 미리보기 등 사용성 보완  

- **추진 실적**  
  - 데이터 처리 자동화 도구 20종, 데이터 추출 자동화 도구 4종 개발  
  - 전체 조회수 5,351회 (2026.4.5. 기준) / 도구 1개당 평균 조회수 223회  
  - 조회수 상위 3개: ① PDF 파일 취합 도구(424회), ② 데이터 분석 도구 모음_올인원(423회), ③ 엑셀 병합 손쉽게 하기(396회)  
  - 주요 도구 사용 시 업무 경감 효과:  
    | 도구명 | 비교 업무 | 수작업 (초/클릭) | 도구 사용 (초/클릭) | 시간 단축률 | 클릭 감소율 |  
    | --- | --- | --- | --- | --- | --- |  
    | 한글→PDF 일괄 변환 도우미 | 한글 파일 10개 PDF 변환 | 360초 / 60클릭 | 60초 / 8클릭 | 83.3% | 86.7% |  
    | 한글 파일 취합 도구 | 한글 파일 10개 취합 | 540초 / 90클릭 | 50초 / 8클릭 | 90.7% | 91.1% |  
    | 다형식 파일 PDF 통합 도구 | 다형식 파일 10개 PDF 통합 | 780초 / 120클릭 | 95초 / 10클릭 | 87.8% | 91.7% |  

#### 2) 교육행정데이터통합관리시스템(교데통) 데이터시각화 사용자매뉴얼 제작 및 통합 AI 매뉴얼 챗봇 구축  
- **추진 배경**  
  - 교데통은 교육행정데이터 분석·시각화 플랫폼이나, 데이터시각화 분야 사용자매뉴얼 부재  
  - 사용자교육, 문의 대응, 오류 안내를 하나의 지원체계로 정리할 필요  

- **자료 정리 및 초안 작성**  
  - 데이터시각화 사용자교육 자료, 강의안, 사용자지원 Q&A, 담당자 설명 등을 바탕으로 생성형 AI 활용 초안 작성 (목차, 본문, FAQ, 오류 대응 문안, 용어 정리)  

- **검토 및 확정**  
  - 담당자 및 교육행정데이터 리더 1~3차 검토 후 누락 보완, 현장 용어 반영, FAQ 품질 보완  
  - 화면 캡처, 마스킹, 강조 표시 반영 후 최종 편집  

- **챗봇 구축 및 운영 방향**  
  - 데이터취합 분야: 2026.1. 시범운영 개시  
  - 데이터시각화 분야 포함 통합 AI 챗봇: 2026.4월부터 확대 추진  
  - 매뉴얼을 근거자료로 활용하여 절차, 자주 묻는 질문, 오류 상황을 상시 확인할 수 있는 구조로 추진  

### 나. AI·데이터 분석 관련 업무수행 경험  
#### 1) AI(생성형 AI) 업무활용 경험  
- 생성형 AI를 활용하여 데이터 분석 자동화 도구 개발, 사용자매뉴얼 작성, AI 매뉴얼 챗봇 구축 업무 수행  
- 데이터 분석 자동화 도구 개발 시 반복적 업무(엑셀 병합, 데이터 품질 점검, 학교명 표준화 등)를 자동화 대상으로 정리, 기능 요구사항 정리, 화면 구성, 코드 초안 작성, 오류 수정, 검토의견 반영까지 직접 수행  
- 기능 확대: 형태 변환·언피벗, 엑셀 분할, 파일명 정리, 이수증 PDF 자동 취합 등  
- 교데통 관련 업무에서는 데이터시각화 사용자교육 자료, 강의안, 사용자지원 Q&A, 담당자 설명을 바탕으로 생성형 AI 활용 사용자매뉴얼 초안, FAQ, 오류 대응 문안 작성 후 통합 AI 매뉴얼 챗봇 구축으로 연결  

#### 2) 데이터 기반 정책 기획 경험  
- 교육청 내 데이터·도구·분석자료·시각화자료 제공체계 운영 업무 수행  
- 부서별 데이터 목록화 및 공유 확대 업무에서 부서별 데이터 현황 분석, 공유범위 재정비, 표준화 양식 마련, 분기별 현행화, 데이터 카탈로그 제작 계획 수립  
- 2025년 기준 부서별 데이터 목록 384개 정리, 2026년 500개 이상 발굴 및 부서 외 공유 비율 확대 목표  
- 교육데이터 허브 운영 과정에서 게시글 표준화, 공개 범위 차등 운영, 자료실 내부·공개 이원화, 대표 홈페이지 이관 등을 추진하여 데이터 접근성과 활용성 향상

## 3. 실전 AX 과제 기획서  
### 1. 과제 개요  
#### 가. 과제명  
- 생성형 AI 기반 공문서 초안 작성 및 한글 서식 자동화 서비스 구축  

#### 나. 추진배경  
- 생성형 AI로 문서 초안 작성은 가능하나, 실제 행정업무에서는 초안 생성 이후 서식 정리 과정이 별도로 필요  
- 현재 실무 현장에서는 다음과 같은 문제 존재:  
  ① 생성형 AI 결과가 기관 공문서 체계와 완전히 일치하지 않음  
  ② 한글 서식은 사람이 다시 손봐야 함  
  ③ 초안 작성과 최종 문서화가 분리되어 작업 흐름이 끊김  
  ④ 복사·붙여넣기·서식 프로그램 실행 등 추가 절차 필요  
  ⑤ 사용자별 문서 형식과 완성도 차이 발생  

#### 다. 목표  
- 생성형 AI가 만든 초안을 실제 제출 가능한 공문서 형식으로 빠르고 일관되게 전환  

### 2. 현안 문제 및 데이터 현황  
#### 가. 문제정의  
- 생성형 AI 문서 초안 작성 가능, 그러나 서식 정리 과정이 별도로 필요  
- 핵심 문제: 생성형 AI 초안을 실제 제출 가능한 공문서 형식으로 얼마나 빠르고 일관되게 전환할 수 있는가  

#### 나. 보유 데이터  
1) 즉시 활용 가능 데이터  
- 공문서 작성용 프롬프트  
- 공문서 체계에 맞춘 문서 작성 기준  
- 한글 서식 자동 편집 프로그램  
- 보고서·계획서 등 공문서 형식 예시  

2) 활용 가능 데이터 성격  
- 기관에서 실제 사용하는 공문서 형식 및 작성 관행  
- 문서 제목, 본문 구조, 소제목 체계, 본문 들여쓰기, 참고 문구 등 서식 요소  

3) 현재 확보 수준  
- 생성형 AI 공문서 초안 작성 프롬프트 정리 완료  
- 생성된 초안을 한글 서식으로 자동 편집하는 로컬 프로그램 제작 완료  

4) 향후 추가 활용 예정 자료  
- 문서 유형별 서식 패턴  
- 자주 작성하는 보고서·계획서 문안 예시  
- 사용자 의견 및 오류 사례  

### 3. 구현 방안  
| 단계 | 내용 |  
| --- | --- |  
| 1단계 초안 작성 기능 구현 | - 사용자가 문서 목적, 주제, 핵심 내용, 문서 유형 입력 - 생성형 AI가 기관 공문서 체계에 맞는 초안 작성 - 문서 유형별 작성 프롬프트 적용하여 초안 품질 확보 |  
| 2단계 공문서 구조 정리 | - 생성된 초안을 공문서 체계에 맞게 자동 분류 - 제목, 소제목, 본문, 참고 문구 등 구분 - 기관 문서 구조에 맞는 형식으로 정리 |  
| 3단계 한글 서식 자동 변환 | - 로컬 기반 한글 서식 자동 편집 로직 활용 - 생성된 초안을 한글 서식 문서로 자동 변환 - 사용자는 별도 복사·붙여넣기 없이 결과 파일 바로 다운로드 |  
| 4단계 웹 기반 통합 서비스 구현 | - 초안 작성과 서식 변환을 하나의 웹페이지에서 처리 - 사용자가 별도 프로그램 실행 없이 문서 작성 → 서식 변환 → 다운로드까지 한 번에 처리 |  
| 5단계 문서 유형별 확장 | - 초기에는 보고서, 계획서 등 활용 빈도가 높은 문서부터 적용 - 이후 결과보고서, 검토자료, 회의자료 등으로 확장 - 자주 사용하는 문서 유형별 템플릿과 서식 기준 추가 반영 |  

#### 가. 적용 기술 방향  
- 생성형 AI를 활용한 초안 작성  
- 한글 서식 자동화 로직 결합  
- 웹기반 사용자 인터페이스 구현  
- 향후 보안 환경을 고려하여 로컬 또는 내부망 적용 가능성 검토  

#### 나. 행정 절차 및 운영 방향  
- 실제 문서 작성이 많은 업무를 중심으로 시범 적용  
- 사용자 의견을 반영하여 프롬프트와 서식 규칙 지속 보완  
- 최종 문서는 담당자가 검토 후 확정하는 방식으로 운영  
- 문서 품질과 서식 일관성을 동시에 확보하는 방향으로 운영  

### 4. 현장 이행계획 및 기대효과  
#### 가. 현장 이행계획  
| 단계 | 내용 |  
| --- | --- |  
| 1단계 | - 현재 보유 중인 공문서 작성 프롬프트와 한글 서식 자동 편집 프로그램 기능 정리 - 초안 작성 기능과 서식 변환 기능의 연결 구조 설계 |  
| 2단계 | - 웹페이지에서 초안 작성과 서식 변환이 함께 이루어지는 시범 버전 구현 - 보고서, 계획서 등 우선 적용 문서 유형 선정 |  
| 3단계 | - 실제 사용자 대상 시범 적용 - 문서 유형별 오류, 서식 불일치, 표현 문제 등 점검 - 사용자 의견을 반영한 기능 보완 |  
| 4단계 | - 자주 사용하는 공문서 유형 중심으로 기능 확대 - 서식 기준과 문서 유형을 추가 반영하여 활용 범위 확대 - 필요시 내부망·로컬 환경 적용 가능성 검토 |  

#### 나. 기대 효과  
- 생성형 AI 초안 작성 이후 별도 복사·붙여넣기 과정 축소  
- 공문서 서식 수정에 들어가는 시간 단축  
- 문서 작성부터 최종 한글 파일 생성까지의 작업 흐름 단순화  
- 사용자별 문서 형식 차이 감소  
- 공문서 작성 부담 경감 및 작성 속도 향상  
- 기존 로컬 서식 자동화 프로그램을 실제 업무 흐름에 맞게 고도화  
- 향후 다양한 공문서 작성 업무로 확장 가능한 기반 마련

## 4. 데이터기반행정 거점 리더 활동 계획  
### 가. 데이터 분석·AI 활용 인재양성 및 문화확산  
- 교직원이 데이터와 AI를 실제 업무에 바로 적용할 수 있도록 사례 중심 경험 공유  
- 데이터 시각화 교육 경험을 바탕으로 실습과 활용 사례 중심 교육 진행  
- 직접 개발한 자동화 도구, 사용자매뉴얼, AI 챗봇, 보도자료 작성 지원 프로그램 등을 활용하여 AI가 실제 업무를 어떻게 줄여주는지 보여줌  
- 신규 사용자와 비전공자도 쉽게 따라올 수 있도록 FAQ, 오류 사례, 활용 예시 중심의 지원자료 지속 보완  

### 나. 데이터기반 정책 수립·제도 마련  
- 부서별 데이터 현황 분석, 공유범위 재정비, 표준화 양식 마련, 분기별 현행화, 데이터 카탈로그 제작 계획 수립 경험 바탕으로 데이터 기반 정책 수립  
- 데이터별 생성 주기, 주요 항목, 활용 목적, 공유 범위, 제한 사유 등을 명확히 정리하여 관리기준 일관성 확보  
- 분기별 현행화와 품질 점검을 통해 누락 데이터, 중복 데이터, 활용 가능한 데이터 지속 정비  
- 공개 가능 데이터, 기관 내 공유 가능 데이터, 제한이 필요한 데이터를 구분하여 데이터 활용 범위 확대  

### 다. 데이터 공유·협력 강화  
- 교육청 내 필요한 데이터, 자동화 도구, 분석자료, 시각화자료를 함께 활용할 수 있는 온라인 제공체계 강화  
- 각 부서 데이터의 명칭, 내용, 생성 주기, 활용 목적, 공유 범위 등을 한눈에 볼 수 있도록 안내자료 제작  
- 공유가 어려운 데이터도 집계자료, 비식별 자료, 항목 축약본 등으로 전환 가능성 검토하여 활용 범위 확대  
- 데이터, 자동화 도구, 분석자료, 시각화자료를 연결하여 실제 업무 활용도 향상  

### 라. AI 전환 거점 리더로서의 활동 방향  
- 교육 과정에서 배운 데이터 분석, 업무자동화, 웹앱, 에이전트 관련 내용을 실제 행정업무 개선 과제와 연결  
- 작은 자동화 사례라도 실제 효과가 확인되면 축적·공유하여 다른 업무로 확산  
- 교육 이수 후 배운 내용을 바로 업무에 적용하고, 적용 결과를 개선과 확산으로 이어감  
- 분석 결과 제시뿐 아니라 실제 업무가 바뀌는 방향으로 연결  

### 2. 과정 참여 필요에 대해 자유롭게 기술  
- 생성형 AI를 활용한 업무자동화 도구 개발, 사용자매뉴얼 및 AI 챗봇 구축, 데이터 공유 확대 계획 수립 등 여러 업무를 직접 추진하며, AI와 데이터 분석이 실제 행정업무 개선에 활용될 수 있음을 확인  
- 현재 실무 경험과 자체 학습을 바탕으로 업무를 추진하고 있으나, 데이터 분석 방법론, AI 서비스 설계, 프로젝트 구조화, 구현 수준을 높이기 위한 체계적인 교육 필요  
- 본 과정은 AI 리터러시, 프롬프트, 엑셀, Python, 모델링, 웹앱, 에이전트, 업무자동화 등을 학습하고, 개인 및 팀 프로젝트를 통해 실제 기관 현안을 과제로 구체화하도록 되어 있어 현재 업무 발전에 적합  
- 과정 참여 후 배운 내용을 바로 업무에 적용하여 실제 행정업무 개선 사례로 이어갈 계획  
- 소속 기관 내에서 AI와 데이터 활용이 실제 업무 방식 변화로 이어질 수 있도록 사례 축적 및 공유 역할 수행 예정
