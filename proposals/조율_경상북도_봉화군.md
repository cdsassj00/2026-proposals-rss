# 지원서 정리 (마크다운 형식)

## 1. 지원동기

### (1) IT·로컬·공직 경험을 통한 역량 통합
- **IT 기업 경력**: 네이버·다음(카카오)·쿠팡·SK텔레콤에서 12년간 서비스 기획자/PM으로 근무  
  - 다음: 광고 플랫폼 개편 프로젝트(30명 규모) 총괄  
  - 네이버: 마일리지·해피빈 서비스 기획  
  - 쿠팡: UX LAB에서 사용자 조사 방법론 설계·수행  
  - SK텔레콤: GPS 기반 IoT 서비스(T-PET) 기획  
- **로컬 크리에이터 활동**: 전북 완주군에서 4년간 로컬푸드 간식 제조·판매  
  - 지역살이의 가능성 발견, 개인 역할의 구조적 한계 체감 → 공직 진출 결심  
- **공직 진출**: 6개월 만에 봉화군 7급 합격, 2024년 1월 도시재생팀 임용  
  - 도시재생 허브시설 3개소 통합 브랜딩·디지털 플랫폼 구축 담당  

### (2) AI 행정 도입 필요성 체감
- **예산 효율성 문제**: 도시재생 시설 통합 플랫폼 외주 개발 시 3,000~5,000만 원 소요 예상  
  - KPI 미정립 상태에서 추가 예산 확보 어려움  
- **프로토타입 한계 경험**: 농업보조사업 원스톱 서비스 기획 시 작동 프로토타입 미구현으로 실행 좌절  
  - "화면에서 돌아가는 결과물"의 조직 설득력 확인  

### (3) 기대효과
- **모델 확산**: 별도 예산·전담 조직 없이 AI를 행정에 도입한 실증 사례 구축 → 전국 군 단위 지자체 확산  
- **번역자 역할**: IT 민간 경력 + 로컬 경험 + 공직 이해를 결합해 기술과 행정 간 중재자 역할 수행  

### 관련 교육 경험
- **IT 기업 시절**: UX 리서치(정성·정량 분석), 서비스 기획 프레임워크, Agile/Scrum 교육 이수  
- **공직 이후**: 지방자치인재개발원 생성형 AI 활용 교육 참여  
- **현재 역량**: Claude·ChatGPT 프롬프트 엔지니어링 심화 활용, Python·SQL·ADSP 학습 중

## 2. 경력 및 실적

### AI·데이터 분석 프로젝트 경험
1. **도시재생 허브시설 통합 플랫폼 기획 (MODI 프로젝트)**  
   - 3개 시설(해오름센터 등) 통합 브랜딩·디지털 플랫폼 구축 PM  
   - 생성형 AI로 RFP 설계·평가기준표 작성 → 문서 효율성 개선 및 기획 품질 향상 경험  

2. **농업보조사업 원스톱 서비스 기획**  
   - 농업인·공무원 불편 해소를 위한 온라인 신청·추적 시스템 기획  
   - 부처 간 칸막이·프로토타입 부재로 실행 좌절 → 기술 역량 필요성 절감  

3. **치유산업·공무원 위키 시스템 제안**  
   - 사내 제안대회에서 치유산업 최우수상·공무원 위키 금상 수상  
   - 문제 구조화 → 해결 방안 제안 가능성 인정받으나, 작동 서비스 미구현  

### AI·데이터 분석 업무 경험
- **생성형 AI 활용**: 사업계획서·RFP·행정 이메일 톤 조율 등에 AI 적용  
- **프롬프트 엔지니어링**: 역할 프롬프팅·스킬 활용으로 행정 문서 품질 관리  
- **사용자 중심 접근법**: 쿠팡 UX LAB 경험을 공공행정에 적용 → 데이터 기반 "빠른 구현·검증" 가능  
- **데이터 활용**: 도시재생 참여자 데이터(200~300명) 연령·관심 분야 분석 → 사업 방향 전환 근거 활용

## 3. 실전 AX 과제 기획서

### 1. 과제 개요
- **과제명**: 도시재생 거점시설 공간 데이터 기반 AI 맞춤 추천·운영 최적화 시스템  
- **추진 배경**:  
  - 3개 시설(200억 원 규모) 준공 예정 → 공간 정보 파편화·반복 문의·데이터 부재 문제  
  - 외주 개발 비용(3,000~5,000만 원) 대비 효율적 AI 솔루션 필요성  
- **목표**:  
  - 정량적: 반복 문의 40% 경감, 공간 가동률 60% 달성, 주민 관심사 300건 축적  
  - 정성적: 수요자 중심 행정서비스 설계, 소멸위기 지자체 AI 행정 혁신 선도  

### 2. 현안 문제 및 데이터 현황
- **5대 문제**: 유휴화 원인 파악 불가, 관리 인력 부족, 디지털 소외 환경, 공간 정보 파편화, 수요 불확실성  
- **보유 데이터**:  
  - 즉시 활용 가능: 공간 상세 정보, MODI 프로젝트 기획 자료, 역량강화사업 참여자 데이터  
  - 수집 예정: AI 대화 로그, 주민 관심사 프로필, 공간별 이용 실적  

### 3. 구현 방안
- **시스템 아키텍처**:  
  - 4개 핵심 모듈(AI 추천 엔진, 지식베이스, 대시보드 등) + 확장 모듈(능동형 안내 자동화)  
  - 기술: LangChain, RAG, ChromaDB, Streamlit, n8n  
- **행정 절차**:  
  - 조례 개정(AI 운영 근거), 개인정보 동의 체계, 위탁 운영 계약 반영, 정보화 인프라 구축  

### 4. 현장 이행계획 및 기대효과
- **이행 단계**:  
  - PHASE 1~5(기반 구축 → PoC 개발 → 통합·고도화 → 능동형·정착 → 확산·제도화)  
- **기대효과**:  
  - 정량적: 반복 문의 40% 감소, 가동률 60% 달성  
  - 정성적: 평가 패러다임 전환("시설 건립" → "시설 활용"), 인력 교체 시 서비스 일관성 유지

## 4. 데이터기반행정 거점 리더 활동 계획

### 1. 조직 내 AI 인재양성 및 문화확산
- **AI학습동아리 신설**: 비전공 공무원 대상 실습 중심 교육 → 1년 내 AI 활용 담당자 10명 양성  
- **지식 공유**: 공무원 위키 시스템 사례 확대 → 부서별 업무 지식 체계화  

### 2. 데이터기반 정책 수립·제도 마련
- **데이터 기반 의사결정 실증**: 거점시설 운영 데이터 분석으로 프로그램 조정 근거 마련  
- **지침 제정**: "봉화군 AI 활용 행정업무 효율화 지침" 추진  

### 3. 외부 데이터 연계 및 협력 강화
- **GBinPLUS+ 플랫폼 연계**: 유동인구·정주환경 데이터로 관광 수요 예측  
- **공모전 출품**: 행안부/NIA 공모에 사례 제출 → 타 지자체와 노하우 공유  

### 과정 참여 필요성
- **강점**: 현장 문제 발견력, 새로운 도구 빠른 습득력  
- **약점**: 직접 구현 기술 부족 → 본 과정을 통해 프로토타입 구현 역량 보완 목표  
- **기대**: 봉화군 현장에서 즉시 활용 가능한 결과물 도출
