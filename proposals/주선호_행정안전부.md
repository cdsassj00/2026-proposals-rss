# 1. 지원동기  
- **참여동기 및 기대효과**  
  소방행정 현장에서 20년 이상 근무하며 화재예방, 정보통신, 재난대응 업무를 수행해 왔습니다. 최근 소방 분야에서도 데이터 기반 의사결정의 중요성이 부각되고 있으나, 현장에서는 여전히 경험과 직관에 의존한 업무 수행이 주를 이루고 있습니다. 특히 화재발생 예측, 취약대상 분석, 소방력 배치 최적화 등에서 AI·빅데이터 분석 기술을 체계적으로 적용할 수 있는 전문 역량이 부족한 실정입니다.  
  저는 전자공학과(정보통신 분야)를 전공하고, 2~3년간 민간에서 웹 백엔드 개발 경험을 쌓았으며, 현재 Python을 활용한 데이터 분석과 AI 모델 개발을 독학으로 수행하고 있습니다. 본 양성과정을 통해 체계적인 AI·빅데이터 분석 방법론을 습득하고, 이를 소방행정 현장에 실질적으로 적용할 수 있는 역량을 갖추고자 지원합니다.  

- **기대효과**  
  1. **AI 기반 화재발생 예측 시스템 구축**  
     기상 데이터, 건축물 정보, 과거 화재 이력 등을 종합 분석하여 화재 취약 시간대·지역을 사전 예측하는 모델 개발 및 현장 인력 배치 최적화를 통한 선제적 재난 대응 체계 구축  
  2. **데이터 기반 의사결정 문화 확산**  
     양성과정에서 습득한 지식을 소속기관 내 교육·전파하여 조직 전체의 디지털 전환(DX) 견인  
  3. **공공부문 AI 전문가 성장**  
     정보통신기술사 자격 취득 학습과 병행하여 이론과 실무를 겸비한 공공부문 AI 전문가로 성장  

- **관련 교육 경험**  
  - 전자공학과 전공  
  - 웹 백엔드 개발 실무 경험(2~3년, 자체 전자상거래 플랫폼 풀스택 개발)  
  - Python 프로그래밍(데이터 분석, 웹크롤링, API 개발 독학 및 실습)  
  - GO 프로그래밍 언어 학습중(시스템 프로그래밍, 임베디드 프로그래밍 역량 확장)  
  - AI 도구 활용 실무: Claude API, OpenAI API를 활용한 LLM 기반 애플리케이션 개발 경험  
  - 소방공무원 대상 AI 활용 강의 수행 경험(소방공무원 AI 활용 역량 강화 교육)  
  - 정보통신기술사 시험 준비중(ICT 전문 자격)

# 2. 경력 및 실적  
- **AI·데이터 분석 프로젝트 경험**  
  1. **AI 기반 실시간 화재발생 예측 대시보드 개발(진행중)**  
     - 개요: 전국소방본부 범용 사용 가능, 화재발생을 AI로 실시간 예측하는 대시보드 플랫폼 개발  
     - 기술 스택: Next.js(프론트엔드) + FastAPI(백엔드) + SQLite3(로컬DB)  
     - 분석기법: 통계적 분석, 머신러닝, 딥러닝 등 33개 분석 방법론 적용  
     - 데이터 소스: 기상청 API, 소방청 화재통계, 건축물 대장 정보 등  

  2. **AI 산업안전관리 플랫폼 'MOS/OSOS' 개발(진행중)**  
     - 개요: 클라우드 기반(SaaS) 산업안전관리 플랫폼 1인 개발  
     - 핵심기능: AI 기반 안전문서 자동생성, 3단계 전사서명 시스템, AI 사업장 관리  
     - 데이터 수집: 웹크롤링, PyMuPDF PDF 파싱, APScheduler 자동화  

  3. **소방 빅데이터 분석 프로젝트**  
     - 화재취약대상 빅데이터 분석 수행(과거 화재 데이터 패턴 분석)  
     - AI 기반 화재예방시스템 연구 수행, 논문 작성중(통계 검증 기반)  

- **AI·데이터 분석 관련 업무 수행 경험**  
  - 소방행정 AI 활용 교육 및 전파(소속기관 공무원 대상 강의, AI 활용 가이드 배포)  
  - 데이터 기반 소방업무 수행(화재취약 지역분석 보고서 작성, 기상 데이터와 화재 발생 상관관계 분석)  
  - AI 기반 개발도구 활용 실무(Claude code CLI, LLM API를 활용한 법률문서 분석 시스템 설계)

# 3. 실전 AX 과제 기획서  
## 1. 과제 개요  
- **과제명**: 다변량 시계열 AI 모델과 공간정보(GIS)를 연계한 실시간 화재발생 예측 및 소방력 동적 배치 최적화 시스템 구축  
- **추진배경**:  
  - 현행 소방력 배치는 정적 지표에 기반하여 연 1회 조정되며, 동적 위험요소(시간대, 계절, 기상조건 등)를 반영하지 못함.  
  - 화재예방 활동이 경험과 직관에 의존하며, 신규 위험요인(전기차 화재, 노후건물 등)에 대한 선제적 대응 어려움.  
  - 화재 관련 데이터가 분산 관리되어 통합 분석이 불가능한 상황.  

- **목표**  
  - **정량적 목표**: 화재발생예측 정확도(F1-score) 75% 이상 달성, 골든타임(평균 7분 → 6분 이내) 15% 단축, 화재피해액 연간 10% 감소  
  - **정성적 목표**: 데이터 기반 스마트 소방행정 패러다임 전환, 전국 소방서 확산 가능한 AI 표준 모델 확립  

## 2. 현안 문제 및 데이터 현황  
- **핵심 문제**  
  1. 화재데이터의 품질 문제(결측치, 이상치, 오타 등)  
  2. 정적 소방력 배치의 비효율성  
  3. 사후 대응적 화재예방 활동의 한계  
  4. 분산된 데이터 환경과 분석 역량 부재  

- **보유 데이터**  
  - 소방청 화재통계 데이터, 기상청 기상관측 데이터, 건물대장데이터, 인구·가구 통계, 소방시설점검데이터 등  

- **수집 예정 데이터**  
  - 한국전기안전공사 전기안전 점검이력, 한국가스안전공사 가스시설 점검이력, 교통 데이터, 한국전력공사 지역별 에너지 사용량  

## 3. 구현 방안  
- **기술적 접근방향**  
  1. **전처리 기법 개발 및 데이터 통합 파이프라인 구축**  
     - AI 기반 데이터 정제 및 검증 자동화  
     - 공공데이터포털, 기상자료개발포털, SGIS 등 외부 API 연동  
  2. **AI 예측 모델 개발**  
     - 다변량 시계열 분석 모델(LSTM, Transformer 기반)  
     - 앙상블 모델(XGBoost + LightGBM + Neural Network)  
  3. **대시보드 및 의사결정 지원시스템**  
     - 실시간 화재 위험 예측 지도(GIS 기반 히트맵)  
     - 소방력 최적 배치 시뮬레이션  

- **제도 개선 및 행정절차**  
  - 부서 협업체계(소방본부 예방안전과/현장대응단/119종합상황실 합동 추진)  
  - 외부기관 데이터 공유 협력체결(기상청, 한전, 한국전기안전공사 등)  
  - 'AI 화재예측 시스템 운영지침' 마련  

## 4. 현장 이행계획 및 기대효과  
- **이행계획**  
  - 기반구축(M+3): 데이터 수집 파이프라인 구축, 외부 기관 협약 추진  
  - 모델 개발 및 검증(M+6): AI 예측 모델 개발 및 백테스팅  
  - 시범운영(M+9): 인천소방본부 119종합상황실 시범운영  
  - 확산 및 제도화(M+12): 인천시 소방서 전체 확대, 운영 지침 공식 시행  

- **기대효과**  
  - 예산절감: 골든타임 축소(10~20% 감소), 화재피해액 감소(연간 10% 절감)  
  - 업무 시간 단축: 화재위험 분석 보고서 작성 시간 90% 단축  
  - 대국민 서비스 품질 향상: 골든타임 확보율 향상(85% 목표), 화재발생 건수 감소(연간 5~10%)
