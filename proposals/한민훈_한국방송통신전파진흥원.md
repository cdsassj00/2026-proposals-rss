# - 업무 활용계획

교육에서 습득한 기술은 첫째, 재난 데이터 표준화와 품질관리 체계 설계에 적용
하겠습니다. 둘째, 상황보고·법령·SOP·사례를 결합한 RAG 기반 지식체계를 구축하여
근거 중심 보고와 의사결정 지원에 활용하겠습니다. 셋째, 119 신고·영상·센서·기상
등 이질 데이터의 통합 분석을 위한 파일럿 과제를 설계하겠습니다. 넷째, 현장과 상
황실에서 반복되는 보고·점검·체크리스트 업무를 프롬프트 기반 업무앱으로 전환할
수 있는 기반을 마련하겠습니다. 다섯째, 교육 종료 후에는 부서 내 실습 공유와 과
제 전파를 통해 기관 내 AI 활용 문화를 확산하겠습니다.

# 2. 실전 AX 과제 기획서

| 1. 과제 개요 |
| --- |
| ○ 과제명 : RAG 기반 근거중심 소방 상황보고·유사사례 추천 시스템 구축 ○ 추진배경 현행 소방 업무는 다수 시스템과 엑셀 관리가 병존하여 상황정보 수집, 유사사례 검색, 법령·SOP 확인, 보고서 작성이 분절적으로 수행되고 있습니다. 복합재난 상황 에서는 여러 출처의 정보를 짧은 시간 안에 종합해야 하나, 현재 방식은 정보 분절 과 결정 지연을 유발하고 보고의 근거 추적도 쉽지 않습니다. 소방OS가 지향하는 "인지→결정→지시→조치→기록" 흐름을 실무에서 가시화하기 위해서는, 먼저 상황 실과 현장에서 즉시 활용 가능한 근거중심 보고 자동화 과제를 구현할 필요가 있습 니다. ○ 목표 본 과제의 목표는 첫째, 상황보고 초안 작성시간을 50% 이상 단축하고 둘째, 유 사사례·규정 탐색시간을 60% 이상 단축하며 셋째, 보고 문안의 근거표시율을 100% 확보하는 것입니다. 아울러 상황실-현장 간 정보 공유의 일관성을 높이고, 향후 재 난대응 OS 확장을 위한 데이터·지식체계 기반을 마련하고자 합니다. |
| 2. 현안 문제 및 데이터 현황 |
| ○ 문제정의 현재 재난 발생 시 담당자는 119 신고 내용, 출동기록, 상황보고, SOP, 법령, 과거 사례, 외부 기상·교통 정보를 여러 시스템과 문서에서 각각 확인해야 합니다. 이 과 정에서 동일 사건에 대한 명칭·분류·자원 정보가 상이하게 기록되고, 보고자가 바뀔 때마다 문안 품질과 판단 기준이 달라질 수 있습니다. 결과적으로 상황판단의 일관 성과 보고 속도가 저하되며, 사후 감사나 학습을 위한 근거 추적도 어렵습니다. 핵 심 문제는 데이터와 문서가 분절되어 있고, 이를 하나의 의미 체계와 근거 기반 응 답 구조로 연결하는 도구가 부재하다는 점입니다. ○ 보유 데이터 즉시 활용 가능 또는 연계 검토 대상 데이터는 ① 내부 데이터: 119 신고 및 출 동기록, 상황보고서, 점검이력, 과거 사건보고서, 화재조사 자료, 내부 SOP·매뉴얼 ② 외부·공공 데이터: 건축물 정보, 기상·산림·교통 데이터, 인구·취약계층 정보, 지역 시 설물 정보 ③ 지식 데이터: 법령, 지침, 사례집, 통계자료입니다. 초기 단계에서는 문 서·상황보고·SOP 중심으로 PoC를 구성하고, 이후 정형 데이터와 외부 API 연계를 확대할 계획입니다. |

| 3. 구현 방안 |
| --- |
| 사건-대상-자원-조치-결과 체계를 기준으로 핵심 메타데이터를 정리하여 보고문 서와 사건데이터를 연결할 수 있는 기본 스키마를 설계하겠습니다. 그 위에 SOP, 법령, 과거 사례, 통계자료를 수집·정제한 RAG 지식베이스를 구축하고, 사용자가 사 건 개요를 입력하면 유사사례, 관련 SOP, 법령 근거, 통계 정보를 함께 제시하면서 상황보고 초안을 자동 생성하도록 하겠습니다. 정형데이터 조회는 Text-to-SQL 가드레일을 적용하여 출동건수, 유사유형 통계, 시간대별 패턴 등을 안전하게 조회하도록 설계하겠습니다. 또한 최종 의사결정은 담당자가 확인하는 Human-in-the-loop 구조로 두고, 권고 근거·모델버전·사용자 수 정내역을 함께 저장해 감사 가능성을 확보하겠습니다. 제도적으로는 상황실, 정보화 부서, 정책기획부서가 참여하는 협업 체계를 마련하고, 문서 표준·접근권한·사용지침 을 정비하겠습니다. |
| 4. 현장 이행계획 및 기대효과 |
| ○ 현장 이행계획 1단계에서는 대상 업무 선정, 데이터·문서 목록화, 핵심 사용자 인터뷰를 통해 PoC 범위를 확정하겠습니다. 2단계에서는 SOP·사례·법령 지식베이스와 상황보고 초 안 생성 프로토타입을 구축하고, 3단계에서는 상황실 시범 적용을 통해 사용자 피 드백을 반영하겠습니다. 4단계에서는 정형데이터 조회, 외부 API 연계, 감사로그 고 도화, 운영지침 수립을 통해 부서 확산 기반을 마련하겠습니다. ○ 기대 효과 본 과제가 도입되면 상황보고 작성과 자료 탐색 시간이 단축되어 담당자의 반복 업무 부담이 감소하고, 근거가 포함된 보고·권고 체계를 통해 의사결정의 일관성과 책임성이 강화될 것입니다. 또한 현장과 상황실이 동일한 기준과 용어를 사용하게 되어 정보전달 오류를 줄일 수 있으며, 향후 멀티모달 인지, 이상징후 탐지, 자원배 치 AI 등 소방OS 상위 과제로 확장할 수 있는 데이터·지식 인프라를 확보할 수 있 습니다. |

# □ AI · 데이터 거점리더 활동계획

| 1. 데이터기반행정 거점 리더 활동 계획 |
| --- |
| 교육 수료 후에는 소속기관 내 AI·데이터 활용 거점리더로서 다음 역할을 수행하 겠습니다. 첫째, 소방 데이터와 문서의 표준화 과제를 발굴하고, 내·외부 데이터 연 계 시 필요한 API·데이터플랫폼 활용계획을 정리하겠습니다. 둘째, 생성형 AI와 RAG 를 활용한 근거중심 보고·분석 업무의 표준 모델을 제시하고, 현업 담당자가 즉시 활용할 수 있는 프롬프트 템플릿과 활용 가이드를 공유하겠습니다. 셋째, 부서 내 소규모 교육과 실습 세션을 운영하여 데이터 활용 문화를 확산하겠습니다. 넷째, 시 범과제의 성과를 정리하여 타 부서와 공유하고, 향후 기관 차원의 AI 전환 로드맵과 제도 개선 과제 발굴로 연결하겠습니다. 다섯째, 공공 AI 활용 시 필요한 보안, 권 한, 감사, 책임성 기준이 실제 업무에 내재화되도록 지원하겠습니다. |
| 2. 과정 참여 필요에 대해 자유롭게 기술 |
| 현재 소방 분야의 AI 전환은 단순 서비스 도입이 아니라, 분절된 데이터와 업무를 하나의 운영체계로 연결하는 수준의 역량을 필요로 합니다. 저는 이번 과정을 통해 데이터 분석, 모델 활용, 시스템 기획, 정책 연계, 거버넌스까지 아우르는 실무형 역 량을 확보하고자 합니다. 특히 본 과정은 실제 업무 문제를 AI·데이터로 해결하는 과제를 설계하고 실행하는 데 직접 연결될 수 있다는 점에서 필요성이 큽니다. 교 육에서 습득한 지식은 개인 역량 강화에 그치지 않고, 소속기관의 AI 전환과 데이터 기반행정 확산을 위한 실질적 성과로 이어지도록 하겠습니다. |
