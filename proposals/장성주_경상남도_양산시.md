## 3. 실전 AX 과제 기획서

### 1. 과제 개요
- **과제명**: 공공데이터 기반 현장학습 격차 분석 및 AI·메타버스 체험 통합 플랫폼 구축  
- **추진 배경**:  
  - 전국 현장학습 운영 현황 분산 관리 → 국가 단위 통합 파악 불가  
  - 지역·소득 계층 간 참여 격차 존재 → 정책 수립 근거 미약  
  - 현장학습 기관 정보 파편화 → 탐색 비용 과다  
  - 사전 체험 수단 부재 → 교육 기회 접근성 제한  
  - 안전사고 데이터 비체계적 관리 → 예방 정책 수립 불가  
- **목표**:  
  - (정량) 전국 현장학습 기관 정보 통합 DB 구축, 격차 분석 지표 체계화, 메타버스 플랫폼 서비스 개시  
  - (정성) 데이터 기반 현장학습 정책 의사결정 지원 체계 마련, 교육 기회 격차 해소, 안전 행정 고도화  

### 2. 현안 문제 및 데이터 현황
- **문제 정의**:  
  ① 데이터 분산(행정) → 국가 단위 비교 분석 불가  
  ② 정책 근거 부재(행정) → 지원 정책 설계 한계  
  ③ 정보 접근 비효율(기술) → 탐색·선택 시간 과다  
  ④ 사전 체험 수단 부재 → 교육 효과·접근성 제한  
  ⑤ 안전 데이터 미관리 → 예방 정책 근거 부족  
- **보유 데이터**:  
  | 데이터 종류 | 출처 및 규모 | 활용 가능 여부 |  
  |-------------|--------------|------------------|  
  | 전국 지자체 현장학습 운영 현황 | 자체 수집 (전국 지자체 대상) | 즉시 활용 가능 |  
  | 교육 관련 공개 데이터 | 공공데이터포털 (교육부·교육청 등) | 즉시 활용 가능 |  
  | 현장 이미지·영상 데이터 | 직접 촬영·수집 | 즉시 활용 가능 |  
  | 학교/학생 신청·참여 기록 | 교육청 연계 필요 (민감정보) | 수집 방법 모색 중 |  
  | 현장학습 사고 통계 | 관계기관 협의 필요 (민감정보) | 수집 방법 모색 중 |  

### 3. 구현 방안
- **추진 원칙**: 단계적 구축, 데이터 거버넌스 선행, 운영가능성 우선  
- **주요 구현 단계**:  
  1. **데이터 수집·통합 파이프라인 (MVP 핵심)**  
     - Selenium·BeautifulSoup 기반 웹 스크래핑  
     - 공공데이터포털 Open API 연동  
     - PyMuPDF·pdfplumber로 PDF 파싱, EasyOCR로 이미지 텍스트 추출  
     - PostgreSQL 통합 DB 구축  
     - Airflow DAG로 수집→정제→적재 자동화  
  2. **격차 분석 엔진 및 정책 대시보드**  
     - K-Means·DBSCAN 군집화, XGBoost·Random Forest 예측, Isolation Forest 이상치 탐지  
     - GeoPandas·Folium 기반 전국 단위 격차 히트맵  
     - Plotly Dash 실시간 시각화 대시보드 구축  
  3. **AI 안내 에이전트 (RAG)**  
     - LangChain + ChromaDB 기반 RAG 시스템  
     - GPT-4/Claude API 연동 질의응답 제공  
  4. **3D·메타버스 체험 플랫폼**  
     - COLMAP 포인트 클라우드 → 3D Gaussian Splatting 자동 재구성  
     - MCP 서버 + Blender Python API 연동  
     - Three.js·WebGL로 웹 브라우저 내 렌더링  
  5. **단계별 로드맵**:  
     - 1단계(3~4개월): 데이터 파이프라인 구축 + 기초 분석 대시보드 오픈  
     - 2단계(3개월): RAG 안내 에이전트 도입 + 설명가능성 고도화  
     - 3단계(3~6개월): 3D/메타버스 시범 구축  

### 4. KPI 체계
- **데이터/파이프라인 KPI**: 수집 성공률 ≥ 95%, 일배치 성공률 ≥ 98%  
- **분석/모델 KPI**: 참여율 예측 MAE·RMSE 목표치 설정, 이상탐지 정밀도/재현율 추적  
- **정책/서비스 KPI**: 정책부서 월간 활성 사용자(MAU), RAG 응답 만족도, 메타버스 체험 완료율  

### 5. 데이터 거버넌스·법·윤리 체계
- 교육청·학교안전공제중앙회·유관기관 MOU 체결  
- 내부 지침: 수집 목적·활용 범위·보존/파기·반출통제 명문화  
- 비식별화 표준: 학생/사고 데이터 최소수집·가명/익명 처리  
- 접근통제(RBAC), 감사로그, 권한승인 워크플로우 도입
