# 3. 실전 AX 과제 기획서  
## **1. 과제 개요**  
- **과제명**: 교육행정직 연수 행정 전주기 AI 전환 플랫폼 구축  
- **추진배경**:  
  - 연수 담당자는 강사 정보 관리, 공문 작성, 수당 계산 등 반복 업무에 시간을 소모하며, 연수 기획이라는 핵심 업무에 집중하지 못함.  
  - 10가지 Pain-point(강사 DB 미구축, 공문 수작업, 출결 오류 등)를 AI로 해결.  
- **목표**:  
  - **정량**: 강사비 처리시간 80% 단축, 출결 오류 0%, 연수 신청률 30% 향상.  
  - **정성**: 담당자 수작업 해방 → 연수 품질 기획 집중, 데이터 기반 행정 문화 정착.  

## **2. 현안 문제 및 데이터 현황**  
- **10개 Pain-point**:  
  1. 강사 DB 미구축 → PDF 파일 수동 관리.  
  2. 공문 수작업 → 매번 동일 내용 재작성.  
  3. 섭외 일정 조율 비효율 → 전화 왕복 및 기록 부재.  
  4. 출강동의서 수작업 → 정보 재입력 및 스캔.  
  5. 출강 안내 시스템 부재 → 개별 작성 및 당일 알림.  
  6. 강사비 이중 수작업 → 엑셀·에듀파인 중복 입력.  
  7. 원고료 기준 주관 판정 → 담당자 육안 비교.  
  8. 수기 출결 → 출석부 집계 오류.  
  9. 공지 채널 분산 → 문자·이메일·전화 병행.  
  10. 교육 환류 체계 부재 → 연수 효과 미확인.  

- **보유 데이터**:  
  - 강사카드 PDF, 연수 이수 현황, 강사비 엑셀, 강의자료 전·후 버전, 연수생 신청 데이터, 교직원 인적 현황, 연수 계획서·결과 보고서.  

## **3. 구현 방안**  
| 기능 | 구현 내용 |  
|------|-----------|  
| **[A-1] 강사 DB-AI OCR** | PDF 강사카드에서 정보 추출 후 DB 저장. |  
| **[A-2] 공문 자동 생성-LLM** | 강의 정보 입력 시 AI가 공문 초안 작성. |  
| **[A-3] 예약 페이지-강사 일정 조율** | 강사가 링크로 일정 수락/제안. |  
| **[A-4] 출강동의서 자동완성** | DB 정보 기반 동의서 자동 작성 및 전자서명. |  
| **[A-5] 출강 안내 자동 발송** | 확정 시 자동 메일 발송 및 리마인더. |  
| **[A-6] 강사비 자동 산정 + 원고료 AI 판정** | 강의 시간·단가 입력 시 자동 계산 및 변경률 분석. |  
| **[B-1] QR 출결 관리** | QR 스캔으로 출결 자동 기록. |  
| **[B-2] 통합 공지 플랫폼** | 문자·이메일·앱 동시 발송. |  
| **[B-3] 만족도 조사 자동화** | AI로 주관식 응답 분석 및 개선 포인트 도출. |  
| **[C-1] 연수기획 도우미 (마법사)** | AI가 커리큘럼 초안 및 강사 조합 제안. |  
| **[C-2] 현업 복귀 후 적용도 추적** | 6개월 후 설문 발송 및 심화 과목 추천. |  
| **[C-3] 개별 면담·부서 의견 수렴 도구** | AI가 의견 분류·요약하여 커리큘럼 조정 지원. |  

## **4. 현장 이행계획 및 기대효과**  
- **이행계획**:  
  1. **준비** (1~2개월): 강사카드 OCR 변환, 법적 검토.  
  2. **1차 개발** (3~4개월): 강사DB·공문자동화·예약페이지 등 구현.  
  3. **2차 개발** (5~6개월): QR 출결·통합 공지 등 추가.  
  4. **파일럿** (7~8개월): 시범 적용 및 피드백 수렴.  
  5. **확산** (9개월~): 전국 교육청 확산 및 오픈소스 공유.  

- **기대효과**:  
  - 강사 정보 재활용: DB 즉시 검색 → 무한 재활용.  
  - 섭외·일정 조율: 예약페이지 1회 → 즉시 완결.  
  - 출강동의서: 자동완성+전자서명 → 수작업 0.  
  - 강사비 집행: 자동계산 → 오류 제거.  
  - 원고료 판정: 수치 자동 판정 → 객관성 100%.  
  - 출결 관리: QR 자동 기록 → 오류율 0%.
