# 1. 지원동기  
입사 후 보험금 지급 금액 산정 업무를 거쳐 현재 제도·기획 업무를 4년 이상 수행하며, **"사용자 입장에서 더 나은 방식은 없을까?"**라는 질문을 꾸준히 제기해왔습니다. 시스템 개선에 대한 열망은 있었으나 코딩 역량 부족으로 아이디어를 실행하지 못하고 문서 제안에 그쳐야 했던 한계가 있었습니다.  

이후 AI 도구를 활용한 **바이브코딩**으로 작은 문제를 직접 해결하며, 실제 업무와의 연결에서 오는 희열을 경험했습니다. 이는 전문지식 없이도 문제 해결이 가능하다는 것을 체감한 계기였습니다.  

이 교육과정에 지원하는 이유는 **개인 역량 향상**뿐 아니라, 체계적인 AI·데이터 교육을 통해 기관의 행정·사업 분야에 분석 기반 시각을 도입하고, 비전공자도 문제 해결이 가능함을 증명하기 위함입니다. 이를 통해 동료들에게 **"노력과 방향성만 있다면 누구나 변화를 이끌 수 있다"**는 증거를 보여주고자 합니다.  

**관련 교육 경험**  
- 기관 내 데이터 분석 교육 프로그램(주 2회, 3주 과정)  
- 프롬프트 엔지니어링 기초(패스트캠퍼스)  
- 데이터분석·AI 리터러시, 노코드 데이터분석(행안부 Databus)  
- AI 챔피언 그린레벨 교육 수료 및 레벨 테스트 통과

# 2. 경력 및 실적  
## AI·데이터분석 활용 경력  
1. **우체국보험 지급심사 데이터 분석**  
   - 3개년 데이터를 파워쿼리로 분석해 다빈도 질병코드 및 청구유형 도출  
   - 모바일 앱 청구 프로세스에 **질병코드 목록화 기능** 제안 → 2024년 하반기 시스템 반영  
   - 금융취약계층 고객의 청구 편의성 제고  

2. **반송사유 원인분석 및 자동심사 룰 개선**  
   - 현장 경험을 바탕으로 분석 결과의 실무적 해석 및 개선 방향 제시  
   - 전문 분석 직원과의 협업을 통해 업무 맥락 연결  

## 생성형 AI 활용 경험  
1. **보도자료 제작 GPTs**  
   - 기초 정보 입력만으로 기관 형식에 맞는 보도자료 자동 생성 → 업무 시간 **90% 이상 단축**  

2. **보험금청구심사 도우미 GPTs**  
   - 증상 입력 시 병원진료과별 질병코드·필수서류 안내 → 출시 후 **200회 이상 실사용**

# 3. 실전 AX 과제 기획서  
## 1. 과제 개요  
- **과제명**: 보험심사 종합 도우미  
  - 면책일 계산, 비례보상 산출, 고객 안내장 생성, 약관 질의응답 통합 시스템  
- **추진 배경**:  
  - 잦은 업무순환으로 인한 면책일 계산 혼선, 비례보상 오류, 안내장 품질 편차, 약관 해석 불일치 발생  
  - 담당자 숙련도에 의존하는 구조적 한계 → 균질한 심사 품질 확보 필요  
- **목표**:  
  - (정량적) 면책일·비례보상 계산, 안내장 작성 시간 **50% 이상 단축**  
  - (정성적) 담당자 역량 무관한 균질한 심사 품질 확보, 금융소비자 보호 강화  

## 2. 현안 문제 및 데이터 현황  
- **문제정의**:  
  ① 실손보험 면책일 계산 혼선  
  ② 비례보상 독립책임액 산출 오류  
  ③ 고객 안내장 작성 품질 불균형  
  ④ 약관 기반 심사 판단의 비표준화  
- **보유 데이터**:  
  - 고객 보험상품 정보, 청구·지급 이력, 고객 안내장 사례 및 템플릿, 우체국보험 전 상품 약관  

## 3. 구현 방안  
- **개발 방향**:  
  - 폐쇄망(금융망) 환경에서 외부 API 없이 내부 서버 기반 웹 애플리케이션 구현  
  - 4가지 기능을 단일 인터페이스로 통합하되, 기능별 독립 작동 구조 유지  
- **세부 기능**:  
  1. **실손보험 면책일 자동 계산기**: 상품별 면책 조건 DB화 → 입력 정보 기반 자동 산출  
  2. **비례보상 독립책임액 계산기**: 웹페이지 입력 → 공식 기반 자동 계산  
  3. **고객 안내장 자동 생성기**: 표준화된 문체로 안내장 생성 (로컬 LLM 활용)  
  4. **약관 질의응답 시스템**: RAG 구조 적용 → 자연어 질문 시 약관 근거 제공  

## 4. 현장 이행계획 및 기대효과  
- **이행계획**:  
  1. 면책 조건 DB 구조화, 비례보상 공식 정의, 안내장 유형 분류, 약관 텍스트 색인화  
  2. 기능 1·2 프로토타입 완성 → 현업 시범 제공 및 피드백 수렴  
  3. 로컬 LLM 기반 기능 3·4 PoC 구현 → 통합 인터페이스 구성  
  4. 전 기능 통합 PoC 현업 배포 → 피드백 반영 및 고도화 계획 수립  
  5. 약관 챗봇을 고객 대상 셀프서비스 챗봇으로 확장  
- **기대효과**:  
  - 업무 효율 향상, 심사 품질 균질화, 민원 감소, 고객센터 부담 경감

# 4. 데이터기반행정 거점 리더 활동 계획  
## 1. 데이터기반행정 거점 리더 활동 계획  
- **역할**: 현장 문제 발굴 및 해결을 위한 가교 역할 수행  
- **추진 과제**:  
  1. **비급여 진료비 이상징후 탐지 프로그램**:  
     - 공공데이터포털 API 연계 → 병원급별 비급여 항목 평균 수가 산정 및 이상치 탐지  
     - 조사 업무 자동화 → 선제적·정확한 조사 지원  
  2. **보험 업무 종합정보 웹페이지**:  
     - 업계동향, 보험통계, 상품·규정 변경, 뉴스 자동 수집·업데이트  
     - 정보비대칭 해소 → 전 직원의 동일 정보 기반 업무 수행 환경 구축  
- **활동 방향**:  
  - 소규모 애자일 방식의 상설 과제 해결 그룹 구성 → 동료 참여 유도 및 작은 성공 경험 공유  

## 2. 과정 참여 필요성  
- 기술 언어 장벽으로 인해 문제 해결 불가능했던 경험 → **AI·데이터 역량 습득**이 필수적  
- 동료들의 업무 애로사항을 직접 해결하고, **행정 담당자도 활용 가능한 도구**임을 증명하고자 함  
- 교육과정 이수를 통해 **실질적 문제 해결 사례**를 축적 → 동료들에게 영향력 전파
