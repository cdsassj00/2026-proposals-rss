| ○ 과제명: 재난 분석/보고 자동화 에이전트 구축 ○ 추진배경 - 공공기관의 재난/관측 관련 업무는 데이터 수집, 분석, 결과 검토, 상황 정리, 보고자료 작성 등 다수의 반복 업무로 구성되어 있음 - 특히 분석 결과를 내부 공유용 문서, 보고서, 브리핑 자료, 검토 메모 형태로 재가공하는 과정에 많은 시간이 소요되며, 담당자별 표현 방식과 정리 수준 차이로 인해 문서 품질의 일관성 확보에도 한계가 있음 - 현재 생성형 AI 기술은 문서 요약, 질의응답, 보고서 초안 작성, 반복 업무 자동화 측면에서 높은 활용 가능성을 보이고 있으나, 공공부문에서는 내부 데이터와 업무 절차를 반영한 기관맞춤형 업무지원 체계로의 전환이 제한적임 - 따라서 본 과제는 재난/관측 데이터 분석 결과와 내부 업무자료를 연계하여, 분석자와 실무자의 반복 문서업무를 줄이고 신속한 상황공유와 의사결정 지원이 가능한 재난 분석/보고 자동화 에이전트를 구축하고자 함 - 본 과제는 단순 챗봇 도입이 아니라, 실제 내부 업무 흐름에 맞춘 분석→정리→보고 지원형 AX 과제로 기획함 ○ 목표 - 재난/관측 데이터 분석 결과를 자동으로 정리/요약/문서화하는 업무지원 에이전트 구축 - 내부 기술문서, 업무지침, 과거 보고자료를 기반으로 질의응답과 보고 초안 작성이 가능한 기관 맞춤형 AI 지원체계 마련 - 반복적인 보고자료 작성 시간을 단축하고, 보고 품질의 일관성과 신속성 향상 - 장기적으로는 재난 분야를 시작으로 타 분석/행정 부서까지 확산 가능한 공용 업무자동화 사례 확보 |
| 2. 현안 문제 및 데이터 현황 |
| ○ 문제정의 - 현재 재난/관측 관련 업무에서는 다음과 같은 비효율이 존재함 · 데이터 분석 결과를 사람이 다시 문장화하고 표와 그래프로 정리해야 함 · 유사한 형식의 보고서, 검토자료, 상황요약 문서를 반복 작성함 · 내부 기준, 지침, 과거 사례를 찾아가며 문서를 작성해야 해 시간이 오래 걸림 · 담당자 경험에 따라 문서 품질과 핵심 요약 수준 차이가 발생함 · 단순 자료 정리와 보고 초안 작성에 전문인력이 지속적으로 투입됨 |

| - 특히 분석업무 자체뿐 아니라 분석결과를 행정적으로 활용 가능한 문서로 바꾸는 과정이 중요한데, 이 구간은 자동화 수준이 낮고 표준화도 부족함 - 따라서 본 과제는 데이터 분석 결과와 내부 문서 자산을 연결하여 자동 요약, 보고 초안 작성, 질의응답, 유사사례 검색 등의 기능을 제공하는 업무지원형 에이전트를 구축하고자 함 ○ 보유 데이터 - 재난/관측 관련 시계열 분석 결과 데이터 - 과거 보고서, 검토 메모, 브리핑 자료, 회의자료 등 내부 문서(보안등급 준수) - 기술지침, 업무 매뉴얼, 절차서, 분석 가이드 문서 - 이벤트 메타데이터, 발생시각, 위치, 분류 결과 등 구조화 데이터 - 향후 연계 가능한 내부 지식자료 및 공공 재난 관련 문서 데이터 - 필요 시 API 또는 내부 데이터플랫폼과 연계 가능한 형태의 디지털 문서 자산 |

| 3. 구현 방안 |
| ○ 1단계: 업무 흐름 및 문서 유형 분석 - 재난/관측 혹은 그외 분석 업무에서 자주 생성되는 문서 유형을 정의 - 예: 상황요약, 결과보고, 검토메모, 브리핑 초안, 기술설명 자료 - 문서별 입력값, 필수 항목, 표현 규칙을 정리하여 자동화 범위 설정 ○ 2단계: 데이터/문서 자산 정비 - 과거 보고서, 업무지침, 기술문서를 수집/분류/정제 - 분석 결과표, 메타데이터, 시각화 산출물을 에이전트 입력 구조에 맞게 정비 - 내부 자료 접근권한, 보안등급, 활용범위를 고려한 문서 관리체계 설계 ○ 3단계: AI 에이전트 구축 - 생성형 AI 기반 문서 요약 및 초안 생성 기능 구현 - RAG 구조를 적용해 내부 지침, 과거 사례, 기술자료를 근거로 답변 로직 설계 - 사용자가 특정 이벤트 또는 분석결과를 입력하면 (핵심 요약, 주요 수치 정리, 보고서 초안, 검토 포인트 제안, 유사사례 추천 등)이 가능하도록 구성 - 필요 시 에이전트가 표준 템플릿에 맞춰 자동 문서를 생성하도록 구현 ○ 4단계: 업무자동화 및 프로토타입 구현 - 반복 보고업무를 줄일 수 있는 간단한 웹앱 또는 내부 프로토타입 구현 - 분석자 입력 최소화, 템플릿 선택, 요약문 생성, 보고 초안 다운로드 등 기능 검토 - 업무 담당자가 직접 사용할 수 있는 UI/UX 구성 |

| ○ 5단계: 제도 개선 및 행정 절차 반영 - AI가 생성한 결과물을 초안 및 검토지원 도구로 활용하는 내부 원칙 마련 - 부서 협업을 통해 문서 표준안, 검토 책임, 승인 절차를 정리 - 보안자료, 민감정보, 내부 지침 반영 범위에 대한 기준 수립 - 시범 운영 결과를 바탕으로 단계적 적용 범위 확대 |
| --- |
| 4. 현장 이행계획 및 기대효과 |
| ○ 현장 이행계획 - 1단계(기획) · 현재 재난/관측 분석 업무 중 반복도가 높은 보고/정리 업무를 선정 · 주요 문서유형과 데이터 흐름을 분석하여 자동화 우선순위 설정 · 교육과정 중 개인 프로젝트로 에이전트 구조와 적용 범위를 구체화 - 2단계(프로토타입 개발) · 내부 문서 및 분석결과 샘플을 활용하여 요약/질의응답/문서초안 작성 기능 구현 · 시범 템플릿을 기반으로 실무 적용 가능성 검토 · 분석 담당자 피드백을 통해 출력 형식 및 품질 보완 - 3단계(시범 적용) · 제한된 범위에서 시범 적용하여 문서작성 시간, 활용도, 품질 향상 효과 측정 · 담당자 검토를 전제로 초안 작성 보조 기능부터 우선 도입 · 내부 업무 특성에 맞는 개선사항 반영 - 4단계(확산) · 성과가 확인된 기능부터 부서 공용 도구로 확대 · 향후 재난 외 타 분석부서, 정책기획, 자료정리 업무까지 확장 검토 · 기관 내 AI 활용 문화 확산과 거점 리더 활동에 연계 (다음 장) |

- 장기적으로는 기관 내 분석 결과와 문서자산이 내부 업무지원 시스템과 연동될
수 있도록 플랫폼 기반 활용 방향을 모색하고자 함

# ○ AI/데이터 활용 인재양성 및 문화확산

- - 부서 내 실무자를 대상으로 AI/데이터 활용 가능 사례를 쉽게 설명하고
- 공유하여, AI를 일부 개발자만의 도구가 아닌 실무 보조 도구로 인식할 수
- 있도록 지원

- - 직접 수행한 분석 사례, 자동화 사례, 프로젝트 경험을 바탕으로 내부 설명,
- 사례 공유, 간단한 실습형 전파 활동을 추진

- - 대학교 강의 경험과 업무 외 비영리 데이터 분석 프로젝트 수행 경험을
- 바탕으로, 실무자가 이해하기 쉬운 방식의 확산 활동을 수행하고자 함

- 부서 내 협업 과정에서 AI를 업무에 연결할 수 있는 신규 아이디어 제안과
- 초기 기획 지원 역할을 맡고자 함

- 자산을 정리하고, 기관 차원의 데이터 활용 기반을 넓히고자 함

- 있도록 내부 업무 원칙과 적용 기준 마련에 기여
- - 반복 보고, 자료 검색, 결과 정리, 질의응답 등 자동화 가능한 업무를 구분하고,
- 적용 가능한 범위와 검토 책임을 명확히 하는 방향으로 제도 정착 지원
- - 장기 목표로는 기관 내 AI 활용 기준, 문서 활용 기준, 데이터 관리 및 활용
- 절차 정비에 실무적으로 기여하고자 함

| - 현재 공공부문에서는 AI 기술 자체보다, 이를 실제 업무에 정착시키고 확산할 수 있는 실무형 리더의 역할이 더욱 강조된다고 생각합니다. - 본인은 재난/관측 데이터 분석과 AI 모델 개발 경험을 보유하고 있으나, 이를 기관 내 공용 업무혁신 체계로 확장하기 위해서는 생성형 AI, RAG, 에이전트, 자동화 등 최신 기술을 행정업무와 연결하는 실전형 학습이 필요하다고 판단했습니다. - 특히 본 소속 기관에서도 최근 업무와 연구 전반에 걸쳐 AI 활용을 적극적으로 권장하고 있으며, 부서에서도 본 과정 지원에 대해 긍정적으로 보고 기꺼이 협조하고 있는 분위기입니다. - 이는 본 과정 수료 후 학습 성과를 개인 역량 향상에만 그치지 않고, 실제 부서 업무와 기관 내 AI 활용 확산으로 연결할 수 있는 여건이 마련되어 있다는 의미라고 생각합니다. - 본 과정은 기술교육에 그치지 않고, 개인 및 팀 프로젝트를 통해 실제 기관 현안을 AI 전환 과제로 구체화하도록 구성되어 있어, 제가 추진하고자 하는 재난 분석/보고 자동화 과제를 발전시키는 데 매우 적합하다고 생각합니다. - 또한 단순한 기술 습득이 아니라 1) 기관 현안 기반 과제 기획, 2) 업무 적용 가능성 검토, 3) 부처/기관 간 협업 경험, 4) AI 거점 리더로서의 확산 역할을 함께 강화할 수 있다는 점에서 참여 필요성이 큰 조건입니다. - 교육 수료 후에는 본 과정에서 습득한 지식과 프로젝트 결과를 바탕으로, 소속 기관 내에서 재난 분석 및 보고 자동화 과제 추진, AI 활용 사례 확산, 실무자 협업 지원, 데이터 기반 행정 문화 조성에 실질적으로 기여하고자 합니다. - 마지막으로 공공부문 재직자로서, AI를 단순한 기술 시연을 넘어 행정 효율화와 현장 문제 해결에 연결하는 실행형 리더로 발전시키는 것을 목표로 하고 있으며, 본 과정은 이를 위한 적합한 교육 기회라고 판단합니다. |
