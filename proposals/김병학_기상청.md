# 3. 실전 AX 과제 기획서  
## 1. 과제 개요  
- **과제명**: 재난 분석/보고 자동화 에이전트 구축  
- **추진배경**  
  - 공공기관 재난/관측 업무는 데이터 수집, 분석, 결과 검토, 보고자료 작성 등 반복 업무로 구성  
  - 생성형 AI 기술은 문서 요약, 질의응답, 보고서 초안 작성, 반복 업무 자동화 측면에서 높은 활용 가능성 보유  
  - 본 과제는 재난/관측 데이터 분석 결과와 내부 업무자료를 연계해 분석자·실무자의 반복 문서업무를 줄이고 신속한 상황공유·의사결정 지원이 가능한 에이전트 구축 목표  
- **목표**  
  - 재난/관측 데이터 분석 결과를 자동으로 정리/요약/문서화하는 업무지원 에이전트 구축  
  - 내부 기술문서, 업무지침, 과거 보고자료를 기반으로 질의응답과 보고 초안 작성이 가능한 기관 맞춤형 AI 지원체계 마련  
  - 보고자료 작성 시간 단축, 보고 품질 일관성·신속성 향상  
  - 재난 분야를 시작으로 타 분석/행정 부서까지 확산 가능한 공용 업무자동화 사례 확보  

## 2. 현안 문제 및 데이터 현황  
- **문제정의**  
  - 데이터 분석 결과를 사람이 문장화·표·그래프로 정리해야 함  
  - 유사한 형식의 보고서, 검토자료, 상황요약 문서를 반복 작성  
  - 내부 기준, 지침, 과거 사례를 찾아가며 문서 작성해야 해 시간 소요  
  - 담당자 경험에 따라 문서 품질과 핵심 요약 수준 차이 발생  
- **보유 데이터**  
  - 재난/관측 관련 시계열 분석 결과 데이터  
  - 과거 보고서, 검토 메모, 브리핑 자료, 회의자료 등 내부 문서  
  - 기술지침, 업무 매뉴얼, 절차서, 분석 가이드 문서  
  - 이벤트 메타데이터, 발생시각, 위치, 분류 결과 등 구조화 데이터  
  - 향후 연계 가능한 내부 지식자료 및 공공 재난 관련 문서 데이터  

## 3. 구현 방안  
1. **업무 흐름 및 문서 유형 분석**  
   - 재난/관측 업무에서 자주 생성되는 문서 유형 정의 (상황요약, 결과보고, 검토메모, 브리핑 초안, 기술설명 자료 등)  
   - 문서별 입력값, 필수 항목, 표현 규칙 정리 → 자동화 범위 설정  
2. **데이터/문서 자산 정비**  
   - 과거 보고서, 업무지침, 기술문서 수집/분류/정제  
   - 분석 결과표, 메타데이터, 시각화 산출물을 에이전트 입력 구조에 맞게 정비  
   - 내부 자료 접근권한, 보안등급, 활용범위를 고려한 문서 관리체계 설계  
3. **AI 에이전트 구축**  
   - 생성형 AI 기반 문서 요약 및 초안 생성 기능 구현  
   - RAG 구조로 내부 지침, 과거 사례, 기술자료를 근거로 답변 로직 설계  
   - 사용자 입력 시 핵심 요약, 주요 수치 정리, 보고서 초안, 검토 포인트 제안, 유사사례 추천 기능 제공  
   - 표준 템플릿에 맞춰 자동 문서 생성 구현  
4. **업무자동화 및 프로토타입 구현**  
   - 반복 보고업무를 줄이는 웹앱 또는 내부 프로토타입 구현  
   - 분석자 입력 최소화, 템플릿 선택, 요약문 생성, 보고 초안 다운로드 등 기능 검토  
   - 업무 담당자 직접 사용 가능한 UI/UX 구성  
5. **제도 개선 및 행정 절차 반영**  
   - AI 생성 결과물을 초안 및 검토지원 도구로 활용하는 내부 원칙 마련  
   - 문서 표준안, 검토 책임, 승인 절차 정리  
   - 보안자료, 민감정보, 내부 지침 반영 범위 기준 수립  
   - 시범 운영 결과를 바탕으로 단계적 적용 범위 확대  

## 4. 현장 이행계획 및 기대효과  
- **현장 이행계획**  
  1. **1단계(기획)**: 반복도가 높은 보고/정리 업무 선정 → 문서유형·데이터 흐름 분석 → 자동화 우선순위 설정 → 개인 프로젝트로 에이전트 구조 구체화  
  2. **2단계(프로토타입 개발)**: 내부 문서·결과 샘플 활용해 요약/질의응답/문서초안 작성 기능 구현 → 시범 템플릿 기반 실무 적용 가능성 검토 → 분석 담당자 피드백으로 출력 형식·품질 보완  
  3. **3단계(시범 적용)**: 제한된 범위에서 시범 적용 → 문서작성 시간·활용도·품질 향상 효과 측정 → 초안 작성 보조 기능 우선 도입 → 내부 업무 특성에 맞는 개선사항 반영  
  4. **4단계(확산)**: 성과 확인된 기능부터 부서 공용 도구로 확대 → 재난 외 타 분석부서, 정책기획, 자료정리 업무까지 확장 검토 → 기관 내 AI 활용 문화 확산과 거점 리더 활동 연계  
- **기대 효과**  
  - **업무시간 절감**: 요약, 문서정리, 보고 초안 작성 시간 단축 → 분석 담당자가 핵심 판단과 검토 업무에 집중 가능  
  - **품질 표준화**: 문서 구성, 핵심 메시지, 주요 수치 정리 방식의 일관성 확보 → 담당자 경험 차이에 따른 편차 감소  
  - **의사결정 지원 강화**: 과거 유사사례, 내부 지침, 분석 결과를 연계한 정보 제공 → 상황 인지와 보고 속도 향상  
  - **확산성 확보**: 재난/관측 분야 외에도 다양한 공공기관의 분석/보고 업무에 적용 가능한 공용 AX 사례로 발전 → 생성형 AI, RAG, 업무자동화의 실제 행정 적용 모델 확보

# 4. 데이터기반행정 거점 리더 활동 계획  
## - 소속 기관의 재난/관측 데이터 분석 업무를 중심으로 AI/데이터 기반 업무혁신 사례 발굴 및 확산  
- **AI데이터 활용 과제 발굴 및 확산**  
  - 재난/관측 데이터 분석, 보고자료 작성, 반복 행정업무 등에서 AI 적용 가능 과제 지속 발굴  
  - 우선순위 높은 과제부터 소규모 프로토타입 또는 시범 적용 형태로 추진  
  - 과제별 적용 효과 정리해 부서·기관에 공유 → 타 부서 참고 가능한 AI 활용 우수사례로 축적  
- **데이터 기반 행정 지원 체계 강화**  
  - 기존 업무에서 축적된 분석 결과, 문서, 매뉴얼, 기술자료 등을 재정리해 활용 가능한 데이터 자산으로 관리  
  - 구조화 데이터와 비구조화 문서를 함께 활용할 수 있도록 내부 데이터 활용 체계 고도화  
  - 분석 결과가 보고, 상황판단, 의사결정 지원으로 연결될 수 있도록 데이터 활용 흐름 정비  
  - 외부 공개 데이터, 유관기관 자료, API 연계 가능한 정보자원 검토 → 분석 활용도 향상  
- **API/데이터 플랫폼 활용 계획**  
  - 공공데이터포털 등 공개 API와 유관기관 데이터플랫폼 활용해 내부 분석자료와 결합 방향 검토  
  - 내부 축적자료와 외부 공개데이터를 연계할 수 있도록 데이터 형식, 메타데이터, 적재 구조 표준화 필요성 검토  
  - 반복 수집이 필요한 자료는 자동 수집 및 정제 체계와 연계해 업무 효율성 향상  
  - 장기적으로 기관 내 분석 결과와 문서자산이 내부 업무지원 시스템과 연동될 수 있도록 플랫폼 기반 활용 방향 모색  
- **AI/데이터 활용 인재양성 및 문화확산**  
  - 부서 내 실무자 대상으로 AI/데이터 활용 가능 사례를 쉽게 설명·공유 → AI를 실무 보조 도구로 인식하도록 지원  
  - 직접 수행한 분석 사례, 자동화 사례, 프로젝트 경험을 바탕으로 내부 설명, 사례 공유, 간단한 실습형 전파 활동 추진  
  - 대학교 강의 경험과 비영리 데이터 분석 프로젝트 수행 경험을 바탕으로 실무자가 이해하기 쉬운 방식의 확산 활동 수행  
  - 부서 내 협업 과정에서 AI를 업무에 연결할 수 있는 신규 아이디어 제안과 초기 기획 지원 역할 수행  
- **데이터 공유/협력 강화**  
  - 분석 담당자, 현업 담당자, 정보화/전산 담당자 간 협업 구조 마련 → AI 과제가 일회성 실험에 그치지 않도록 추진  
  - 내부 문서, 데이터, 분석결과, 기술자료의 공유 범위 정리 → 보안 수준을 고려한 활용 기준 검토  
  - 유사한 분석 과제를 수행하는 부서와의 협업을 통해 공통 기능과 공용 자산 정리 → 기관 차원의 데이터 활용 기반 확대  
- **데이터 기반 정책 수립 및 제도 마련 지원**  
  - AI 분석 결과를 의사결정 지원도구로 활용할 수 있도록 내부 업무 원칙과 적용 기준 마련에 기여  
  - 반복 보고, 자료 검색, 결과 정리, 질의응답 등 자동화 가능한 업무 구분 → 적용 가능한 범위와 검토 책임을 명확히 하는 방향으로 제도 정착 지원  
  - 장기 목표: 기관 내 AI 활용 기준, 문서 활용 기준, 데이터 관리 및 활용 절차 정비에 실무적으로 기여
